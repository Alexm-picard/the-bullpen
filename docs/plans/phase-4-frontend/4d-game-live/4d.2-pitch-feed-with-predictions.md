# 4d.2 — Pitch feed with predictions overlay

> Owning phase: `phase-4-frontend/4d-game-live` · Estimated effort: `5–8 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:

- Pitch-by-pitch feed on `/game/:id` showing the last ~50 pitches with model prediction overlay
- For each pitch: count, pitcher, batter, predicted distribution at the moment of release, observed outcome
- Visual treatment: small bar chart per pitch showing the 5-class predicted probabilities + actual outcome marker
- Live updates: TanStack Query merges new rows from `useLivePitches` into the feed

**OUT of scope**:

- Strike-zone heatmap → v1.5
- Game-state-machine internals → 4d.1

**Soft cut**: 4d (this entire sub-tree) is the FOURTH soft-cut candidate; 4d.2 specifically can be cut if 4d.1 is delivered but polish-time is short.

---

## Objectives

1. Feed renders 50 pitches in <500 ms.
2. New pitches appear at the top with a subtle entrance hint (e.g., 200ms fade).
3. Predicted-vs-actual disagreement highlighted (✗ marker in brick-red accent).

## Step-by-step implementation tasks

1. Backend: extend `/v1/games/{id}/pitches` to also return the prediction the API made for each pitch (look up in `prediction_log` by `(game_id, at_bat_index, pitch_number)`):
   ```sql
   SELECT live.*, pl.prediction
   FROM pitches_live live
   LEFT JOIN prediction_log pl
     ON JSONExtractUInt(pl.features,'game_id') = live.game_id AND ...
   WHERE live.game_id = ? AND live.id > ?
   ```
2. Frontend: author `<PitchCard pitch />` — count display in JetBrains Mono tabular; pitcher / batter names; `<ProbabilityBar predicted />`; outcome icon.
3. Author `<ProbabilityBar five-class />` — horizontal stacked bar with 5 segments (out/1B/2B/3B/HR or pitch outcome classes), tooltip on hover showing exact probabilities.
4. Author `<PitchFeed pitches />`:
   - Reverses array (newest first).
   - 200 ms fade for new entries (Framer Motion or CSS transition).
   - Disagreement highlight: thin brick-red rule on the right side.
5. Tests: Vitest for `<PitchCard />` (snapshots for agreement / disagreement); Playwright for the page (50 pitches visible).

## Known edge cases

- **Pitch with no prediction** (model missed the live cutoff): render `n/a` placeholder.
- **Long player names**: truncate with ellipsis on small viewports.
- **Color-blind users**: agreement / disagreement uses both color AND glyph (✓/✗).
- **Live updates during scroll**: don't snap to top on every new pitch; let the user scroll. New-pitch indicator at the top.
