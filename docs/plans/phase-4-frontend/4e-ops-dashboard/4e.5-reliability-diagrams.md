# 4e.5 — Ops: reliability diagrams (per model version)

> Owning phase: `phase-4-frontend/4e-ops-dashboard` · Estimated effort: `4–6 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:

- Reliability tab on `/ops` showing per-model, per-class reliability diagrams
- Sourced from the eval artifact `reliability_diagrams.png` for tree models OR computed live from `prediction_log` for current state
- Side-by-side: at-training vs current-on-real-traffic
- Backend endpoint `GET /v1/ops/reliability/{model_version_id}` returning binned data live + a path to the static training-time PNG

**OUT of scope**:

- Per-park reliability (30 figures) — link to the eval artifact directory rather than render inline
- Per-segment reliability — covered by 4e.2's drift charts

---

## Objectives

1. Reliability tab shows per-class diagrams for the active champion.
2. Switch in the UI to compare against training-time diagram.
3. Loads in <500 ms.

## Step-by-step implementation tasks

1. Backend `GET /v1/ops/reliability/{vid}?source=current|training`:
   - `current`: queries last-7-days `prediction_log` joined with outcomes, returns binned reliability data.
   - `training`: returns the path to the static PNG (frontend fetches the image directly via `/v1/ops/eval-artifact/{vid}/reliability_diagrams.png`).
2. Author static-file controller for `eval/` PNGs (only public read, only specific filenames whitelisted to prevent path traversal).
3. Frontend: `<ReliabilityDiagramGrid />` shows 5 small diagrams (one per class for pitch model; combined for binary) using the same `<ReliabilityDiagram />` component from 4b.3.
4. Toggle between `current` and `training` data sources.
5. Tests: Vitest snapshot; Playwright loads tab.

## Known edge cases

- **Path traversal on static endpoint**: whitelist file names exactly (`reliability_diagrams.png`, `confusion_matrix.png`, `feature_importance.csv`); reject everything else with 404.
- **Low-N current data**: if past 7 days has <500 predictions, render placeholder ("insufficient data — check back later").
- **Per-park (30 figures)**: link to the eval artifact directory; don't render inline (too many).
