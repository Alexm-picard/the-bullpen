## Step-by-step implementation tasks

1. Backend: `GET /v1/ops/drift?model=...&type=...&since=...` queries `drift_metrics` and returns sparse `[{computed_at, feature_or_segment, metric_value, sample_size}, ...]`.
2. Frontend: `<CalibrationTrendChart />` using Recharts:
   - X-axis: time (last 30 days).
   - Y-axis: calibration error.
   - Reference line at training calibration.
   - Reference line at `1.5 × training_cal` (page threshold).
   - Red shading when value > threshold for 3+ days.
3. `<PsiHeatmap />` — features on Y, days on X, color = PSI value (Viridis).
4. Tab section orchestrates filter dropdowns (`modelName`) and stitches charts.
5. Tests: Vitest snapshot of the chart with fixture data; Playwright for page load.

## Known edge cases

- **Off-season**: 30-day window may straddle a no-baseball period. Show a gap with a "no data" annotation.
- **Heatmap density**: 30 features × 30 days = 900 cells. Mantine handles fine.
- **`metric_value=NaN` in time series**: filter out before rendering.
- **Time-zone display**: convert UTC `computed_at` to ET in display.
