# 4e.3 — Ops: A/B status + traffic_pct slider

> Owning phase: `phase-4-frontend/4e-ops-dashboard` · Estimated effort: `4–6 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:

- A/B tab showing current routing config per model_name (champion + challenger + traffic_pct + mode)
- Current `experiment_results` rows (running/passed/failed/aborted)
- A `traffic_pct` slider (admin-only) that updates routing on commit
- Visual: paired metric history (champion vs challenger Brier over time) once a challenger is in shadow/A/B

**OUT of scope**:

- Promotion button — separate admin endpoint accessible from registry section (4e.1)
- Experiment authoring UI — admin posts JSON to backend; UI only shows status

---

## Objectives

1. Page shows live routing config; auto-refreshes every 30 s.
2. Admin slider commits via API; non-admins see slider disabled.
3. Experiment status visible.

## Step-by-step implementation tasks

1. Backend public endpoints:
   - `GET /v1/ops/routing` — list of all `model_routing` rows.
   - `GET /v1/ops/experiments?status=running` — list of experiments.
2. Frontend: `<RoutingConfigCard />` per model showing champion, challenger, mode, traffic_pct.
3. `<AdminTrafficSlider value onCommit />`:
   - Disabled by default for non-admin users (no token).
   - When the user provides admin credentials (via a small modal or local env), enables.
   - On commit, calls `POST /v1/admin/routing/{name}/traffic-pct` with HTTP basic.
4. Auth handling for slider: lightweight — admin enters basic-auth creds in a modal; client stores them in memory only; never persists. (We are the only "admin" anyway.)
5. `<ExperimentStatusList />` — table of running/recent experiments.
6. Paired metric chart: shows champion vs challenger Brier over time using `prediction_log` aggregated daily.
7. Tests: Playwright loads tab, slider visible (disabled).

## Known edge cases

- **No challenger configured**: card shows "No challenger" + "Promote a candidate to shadow first".
- **Concurrent admin slider clicks**: backend cache invalidates; subsequent reads pick up new value.
- **Auth UX**: cleanest: admin opens `/ops` with `?admin=true` query param to expose the auth modal. Otherwise hidden.
