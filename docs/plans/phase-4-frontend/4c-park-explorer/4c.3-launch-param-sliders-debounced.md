# 4c.3 — Launch-param sliders (debounced)

> Owning phase: `phase-4-frontend/4c-park-explorer` · Estimated effort: `4–6 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:

- Three Mantine sliders: launch_speed (60–120 mph), launch_angle (-10–60°), spray_angle (-45–45°)
- Each slider debounced 200 ms before triggering a re-prediction
- Re-prediction calls `/v1/predict/batted-ball/all-parks` and updates the grid
- TanStack Query handles cache so rapid slider changes don't double-fetch

**OUT of scope**:

- Wind / atmosphere sliders — defaulted to standard atmosphere; v1.5 candidate
- Spin-rate slider — most users don't intuit spin; defer to v1.5
- Polish iteration → 4c.4

---

## Objectives

1. Slider changes refetch within 200 ms of user pause.
2. Grid updates smoothly (no flash of empty state mid-fetch).
3. Disabled state during in-flight fetch (subtle, not jarring).

## Step-by-step implementation tasks

1. Author `useDebouncedValue<T>(value: T, delayMs: number): T` (standard pattern).
2. Author `<LaunchParamSliders values onChange />`:
   ```tsx
   <Stack>
     <Slider value={values.launchSpeed} onChange={...} min={60} max={120} step={0.5} label={(v)=>`${v} mph`} />
     <Slider value={values.launchAngle} onChange={...} min={-10} max={60} step={0.5} label={(v)=>`${v}°`} />
     <Slider value={values.sprayAngle} onChange={...} min={-45} max={45} step={1} label={(v)=>`${v}°`} />
   </Stack>
   ```
3. In `<ParksPage />`:
   ```tsx
   const [params, setParams] = useState({
     launchSpeed: 110,
     launchAngle: 28,
     sprayAngle: 0,
   });
   const debounced = useDebouncedValue(params, 200);
   const { data } = useAllParksPrediction({
     ...debounced,
     parkId: "NYA",
     stand: "R",
   });
   ```
   Use `previousData` from TanStack Query to keep last-rendered grid during refetch.
4. Subtle "updating..." indicator on the slider rail during refetch.
5. Tests: e2e moves slider, expects grid to update; no flash of empty state.

---

## Known edge cases

- **Off-edge values (e.g., -10° angle)**: backend validates `-30..60`; client constrains to `-10..60` for usability.
- **Slider on touch devices**: Mantine's slider supports touch. Test on mobile manually in Phase 5.4 (a11y).
- **Network latency**: at 200ms debounce + 100ms fetch, total feedback ≈ 300ms. Acceptable.
