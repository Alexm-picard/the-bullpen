## Scope boundaries

**IN scope**:
- 30 hand-curated SVG outlines of MLB stadium playing fields (foul lines + outfield wall)
- Each SVG sized to a normalized viewBox (e.g., 0–500 ft) so that overlays from `2c.3`'s polyline match
- Static assets under `frontend/public/parks/<park_id>.svg`
- A small index `frontend/src/data/park-meta.json` mapping `park_id` → `{ name, svgPath, fenceColor, ... }`

- The 30-grid heatmap → 4c.2
- Sliders → 4c.3

## Step-by-step implementation tasks

1. For each park, draw the SVG by hand (or trace from MLB published outlines + Andrew Clem's diagrams). Keep it minimal: foul lines, outfield wall as a polyline, dirt/grass omitted. Stroke width 2px, color `currentColor` so consumers can theme.
2. Normalize viewBox: `0 0 500 500` with home plate at the bottom center. 1 unit = 1 foot.
3. Author `park-meta.json`:
   ```json
   {
     "NYA": { "name": "Yankee Stadium", "svgPath": "/parks/NYA.svg", "shortFenceFt": 314, "centerFenceFt": 408 },
     ...
   }
   ```
4. Author `<StadiumSvg parkId children />`:
   ```tsx
   export function StadiumSvg({ parkId, children }: Props) {
     return (
       <svg viewBox="0 0 500 500" role="img" aria-label={`${parkId} field`}>
         <use href={`/parks/${parkId}.svg#field`} />
         <g className="overlay">{children}</g>
       </svg>
     );
   }
   ```
   The SVG file uses `<symbol id="field">` so consumers can `<use>` it inline.
5. Author Vitest test: `<StadiumSvg parkId="NYA" />` renders an SVG element with `aria-label`.
6. Document the curation process in `frontend/public/parks/_README.md` so future park changes are reproducible.

## Known edge cases

- **Hand curation drift from `2c.3`**: the geometry file is authoritative for HR classification; the SVG is just visual. They should agree but minor discrepancies are fine.
- **`<use href>` cross-origin**: hosting from `frontend/public/` is same-origin; works.
- **Accessibility**: every SVG needs `role="img"` + `aria-label`. Don't omit.
- **Field orientations vary**: home plate is always at bottom-center in our coordinate system. Document.
