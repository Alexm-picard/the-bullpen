# 4c.2 — Basic 30-park grid (deliberately not polished)

> Owning phase: `phase-4-frontend/4c-park-explorer` · Estimated effort: `5–7 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:

- A 6×5 grid of stadium thumbnails on `/parks`
- Each thumbnail uses `<StadiumSvg />` from 4c.1 with an HR-probability overlay (single color tint)
- Initial overlay computed from a default canonical input (110 mph / 28° / no wind)
- Backend endpoint `POST /v1/predict/batted-ball/all-parks` returning 30 probabilities in one call
- **Deliberately NOT polished** — that's 4c.4. Build the spine first.

**OUT of scope**:

- Sliders → 4c.3
- Polished heatmap visual → 4c.4

---

## Objectives

1. 30 thumbnails render in a grid in <1 s.
2. Each thumbnail shows P(HR) numerically below + a single-color tint overlay.
3. Endpoint returns 30 predictions in one call (avoids 30 round-trips).

## Step-by-step implementation tasks

1. Backend: author `/v1/predict/batted-ball/all-parks` returning `{ park_id: probs[5] }` for all 30 parks in one ONNX call. The MLP from 2c.5 already produces 30-park output natively.
2. Frontend: hook `useAllParksPrediction(input)`:
   ```ts
   useQuery({
     queryKey: ["parks", "all", input],
     queryFn: () => post("/predict/batted-ball/all-parks", input),
   });
   ```
3. Author `<ParkThumbnail parkId probHr />` — renders `<StadiumSvg />` with an SVG `<rect>` overlay opacity proportional to `probHr`. Park name + numeric P(HR) below.
4. Author `<ParksPage />` — fetches all 30 in one call, renders 6×5 grid. Loading: 30 skeleton tiles. Error: text + retry. Empty: "No predictions available."
5. Tests: Vitest for the thumbnail (snapshot); Playwright for the page (renders 30 tiles).

## Known edge cases

- **30 SVGs in one viewport**: ~30 × 5 KB = 150 KB. Acceptable; lazy-loaded with the route.
- **Color contrast**: pure-tint overlay can clash with editorial tones. Use `colors.viz.viridis` swatches at index 4 for the tint. Polish in 4c.4.
- **Initial input**: hardcoded canonical (110 mph / 28° / center field). Sliders in 4c.3 will replace.
- **30-output ONNX**: already produced by 2c.5; the controller just exposes it.
