## Step-by-step implementation tasks

1. Backend: author `PlayerRepository.search(q, limit)` querying:
   ```sql
   SELECT pitcher AS id, 'pitcher' AS role, count() AS pitches
   FROM pitches FINAL
   WHERE toString(pitcher) LIKE concat('%', ?, '%')
   GROUP BY pitcher
   ORDER BY pitches DESC
   LIMIT ?
   ```
   Note: real player names aren't in `pitches` (just IDs). Add a small `players` lookup table (V017 migration in this leaf) populated nightly from MLB Stats API roster endpoint.
2. V017 — `players` table:
   ```sql
   CREATE TABLE players (
     id UInt32, name String, primary_position FixedString(2), updated_at DateTime
   ) ENGINE = ReplacingMergeTree(updated_at)
   ORDER BY id;
   ```
3. Author `PlayerController`:
   ```java
   @GetMapping("/v1/players/search")
   public List<PlayerSearchResult> search(@RequestParam @Size(min=2,max=50) String q,
                                          @RequestParam(defaultValue="10") @Min(1) @Max(50) int limit) {
     return repo.search(q, limit);
   }
   ```
4. Frontend: author `usePlayerSearch`:
   ```ts
   export const usePlayerSearch = (q: string) =>
     useQuery({
       queryKey: ["players", "search", q],
       queryFn: () =>
         get<PlayerSearchResult[]>(
           `/players/search?q=${encodeURIComponent(q)}`,
         ),
       enabled: q.trim().length >= 2,
       staleTime: 60_000,
     });
   ```
5. Author `<PlayerSearch />` using Mantine `Autocomplete` with debounced input.
6. Wire the page; navigate on selection.
7. Author Playwright test: type "Judge", expect dropdown to show ≥1 result, click first → URL changes to `/players/:id`.

## Known edge cases

- **`players` table empty on first deploy**: search returns empty. Backfill on first-ever Phase 4 deploy via `python -m thebullpen.ingest.players_pull`.
- **Common names** (e.g., "Smith"): too many results. Cap at 10; let users refine.
- **ID-as-string queries**: a recruiter pasting a Statcast ID like "660271" should match. The `LIKE` query handles via `toString(pitcher)`.
- **Race conditions**: rapid typing + slow network; TanStack Query handles cancellation.
