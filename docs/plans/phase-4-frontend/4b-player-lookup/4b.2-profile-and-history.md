## Step-by-step implementation tasks

1. Backend: extend `PlayerController` with:
   ```java
   @GetMapping("/v1/players/{id}/predictions")
   public List<PlayerPredictionRow> predictions(@PathVariable long id, @RequestParam(defaultValue="50") int limit) {
     return predictions.findRecent(id, limit);
   }
   ```
2. Backend `predictions.findRecent(id, limit)`:
   ```sql
   SELECT pl.request_at, pl.model_name, pl.prediction, p.description, p.events
   FROM prediction_log pl
   JOIN pitches p
     ON JSONExtractUInt(pl.features, 'pitcher_id') = ?
        OR JSONExtractUInt(pl.features, 'batter_id') = ?
   WHERE p.game_date = toDate(pl.request_at)
   ORDER BY pl.request_at DESC LIMIT ?
   ```
3. Frontend: author `usePlayerProfile(id)` and the table component.
4. Header: name, "X recent predictions / Y observed outcomes".
5. Loading: skeleton table with 5 rows. Error: text + retry button. Empty: "No recent predictions for this player."
6. Tests: Vitest for the table component (snapshot of 3-row rendering); Playwright covers happy path.

## Known edge cases

- **No predictions for that player**: empty state with link back to search.
- **Joined query performance**: ClickHouse JOIN on JSON-extracted fields is slow. Pre-compute a `pitcher_id` column in `prediction_log` via `MATERIALIZED COLUMN` (alter in this leaf if perf is bad).
- **Truncation**: `prediction` is JSON-encoded; UI parses to show the winner class + p(winner).
