## Step-by-step implementation tasks

1. Backend SQL:
   ```sql
   WITH binned AS (
     SELECT toFloat64(JSONExtractFloat(prediction, 'p_winner')) AS p,
            (CASE WHEN p.events = 'home_run' THEN 1 ELSE 0 END) AS y
     FROM prediction_log pl
     JOIN pitches p ON ...
     WHERE JSONExtractUInt(pl.features,'batter_id') = ?
   )
   SELECT bin, avg(p) AS predicted, avg(y) AS actual, count() AS n
   FROM (SELECT *, floor(p*10)/10 AS bin FROM binned)
   GROUP BY bin ORDER BY bin;
   ```
2. Author `<ReliabilityDiagram bins />`:
   - X-axis: predicted probability (0..1).
   - Y-axis: actual frequency (0..1).
   - Diagonal `y=x` reference (brick-red).
   - Points colored by sample size (Viridis intensity).
3. Hook `usePlayerCalibration(id, model)`.
4. Render on profile page below history table.

## Known edge cases

- **Low-N player**: <50 predictions → render placeholder, not a noisy chart.
- **Multi-class diagram**: per-class only; pick the player's most-common predicted class for the diagram. v1.5 may add multi-class small multiples.
- **Dynamic model selection**: query param `?model=pitch_outcome_pre|post` selects which model's predictions to plot.
