# 3a.4 — Promotion API + admin endpoints

> Owning phase: `phase-3-ml-wrapper/3a-registry` · Estimated effort: `5–7 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:
- HTTP endpoints under `/v1/admin/registry/`:
  - `GET /v1/admin/registry/{model_name}` — list versions
  - `POST /v1/admin/registry/{model_name}/register` — register a new version (proxy to RegistryService)
  - `POST /v1/admin/registry/{model_name}/promote/{version_id}` — transition stage
  - `GET /v1/ops/registry/{model_name}` — read-only public view (for Ops dashboard)
- HTTP basic auth gate on `/v1/admin/*` (Risk Register G10 split)
- Promotion to `champion` requires a passing `experiment_results` row (decision `[72]`)

**OUT of scope**:
- A/B routing config endpoints — `3b.1`
- Snapshot management — `3a.5`

---

## Objectives

1. Admin endpoints functional and gated by HTTP basic.
2. Public read endpoints work without auth.
3. Promotion to champion is rejected if no passing `experiment_results` row exists.
4. OpenAPI spec includes all endpoints; frontend codegen succeeds.

## Step-by-step implementation tasks

1. Replace `SecurityConfig`:
   ```java
   @Configuration
   @EnableWebSecurity
   public class SecurityConfig {
     @Bean
     SecurityFilterChain chain(HttpSecurity http, BullpenAdminBasicAuth ba) throws Exception {
       return http
         .csrf(c -> c.disable())
         .authorizeHttpRequests(a -> a
           .requestMatchers("/v1/admin/**").authenticated()
           .anyRequest().permitAll())
         .httpBasic(Customizer.withDefaults())
         .build();
     }
     @Bean
     UserDetailsService users(@Value("${bullpen.admin.basicauth}") String creds) {
       String[] uv = creds.split(":", 2);
       return new InMemoryUserDetailsManager(
         User.withUsername(uv[0]).password("{noop}" + uv[1]).roles("ADMIN").build());
     }
   }
   ```
   `bullpen.admin.basicauth=${THEBULLPEN_ADMIN_BASIC_AUTH}` in `application.yml` (already in `secrets.env`).
2. Author `RegistryAdminController`:
   ```java
   @PostMapping("/{name}/register")
   public ModelVersion register(@PathVariable String name, @Valid @RequestBody RegisterRequest req) {
     return registry.register(req.withModelName(name));
   }

   @PostMapping("/{name}/promote/{vid}")
   public ModelVersion promote(@PathVariable String name, @PathVariable long vid, @Valid @RequestBody PromoteRequest req) {
     return registry.transitionStage(vid, Stage.valueOf(req.targetStage().toUpperCase()));
   }
   ```
3. Author `RegistryOpsController` with `GET` endpoints — list versions, read individual eval artifact summary. No auth.
4. Modify `RegistryService.transitionStage`:
   ```java
   if (newStage == Stage.CHAMPION) {
     var passingExp = experimentRepo.findPassingExperiment(mv.modelName(), mv.id());
     if (passingExp.isEmpty()) {
       throw new IllegalTransition("No passing experiment_results row for " + mv.modelName() + " v" + mv.version());
     }
   }
   ```
5. Author tests:
   - 401 on `/v1/admin/registry/X` without auth.
   - 200 with valid HTTP basic.
   - Promote to champion without experiment row → 400 `IllegalTransition`.
   - Insert a passing `experiment_results` row, retry → 200.
6. Update OpenAPI; regen frontend types.

## Known edge cases

- **Bootstrap promotion**: when no champion exists yet, the first promotion to `champion` is special — no prior champion to archive. Allow without `experiment_results`? **Decision**: yes, but only when it's the first version registered (otherwise require the gate). Document.
- **Multiple challenger candidates**: registry tracks one champion + one challenger via `model_routing`. To promote a different candidate, demote the current challenger first (or the API can do it implicitly).
- **HTTP basic over Cloudflare Tunnel**: secure (TLS edge); `Cf-Connecting-Ip` header preserved for rate-limit logic.
- **CSRF**: disabled (no cookies, no CSRF surface).
