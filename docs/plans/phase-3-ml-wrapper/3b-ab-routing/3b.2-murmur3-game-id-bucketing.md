# 3b.2 — Murmur3 game_id bucketing

> Owning phase: `phase-3-ml-wrapper/3b-ab-routing` · Estimated effort: `2–3 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:
- `Bucketer.bucket(gameId, modelName)` returns an integer in `[0, 1000)`
- Stable, deterministic — same `(gameId, modelName)` always returns the same bucket
- `route(gameId, modelName, RoutingConfig) -> Role` returns `CHAMPION` or `CHALLENGER` based on the bucket and `traffic_pct`
- For requests without a `gameId` (e.g., Park Explorer hypothetical predictions), fall back to a session UUID stored in a request header

**OUT of scope**:
- Shadow-mode (always-both) dispatch → `3b.3`
- Logging the role → `3b.5`

---

## Objectives

1. `bucket(gameId, modelName)` is deterministic across JVM restarts.
2. Distribution of buckets is uniform on real game IDs (chi-squared test passes).
3. `route(...)` correctly assigns ~`traffic_pct`% of game IDs to challenger over a 1000-bucket sample.

## Step-by-step implementation tasks

1. Author `Bucketer`:
   ```java
   private static final HashFunction H = Hashing.murmur3_32_fixed();

   public int bucket(long gameId, String modelName) {
     return Math.floorMod(
       H.newHasher().putLong(gameId).putString(modelName, StandardCharsets.UTF_8).hash().asInt(),
       1000);
   }

   public Role route(long gameId, String modelName, RoutingConfig cfg) {
     if (cfg.mode() == RoutingMode.SHADOW) return Role.CHAMPION;  // shadow returns champion to user
     int b = bucket(gameId, modelName);
     int threshold = (int) Math.round(cfg.challengerTrafficPct() * 10);  // pct → buckets out of 1000
     return b < threshold ? Role.CHALLENGER : Role.CHAMPION;
   }
   ```
2. Author `BucketerTest`:
   - Determinism: same `(gameId, modelName)` → same bucket across 1000 iterations.
   - Uniformity: for 100K random game IDs, chi-squared GOF on 100-bucket bins passes at α=0.01.
   - 50% routing: with `traffic_pct=50`, ~50% of bucketed IDs go to CHALLENGER.
3. Document the model_name salt: prevents cross-model bucket alignment (a game routed to challenger for `pitch` shouldn't be auto-routed to challenger for `batted_ball`).

## Known edge cases

- **`gameId == 0`**: edge case for missing game. The Park Explorer / hypothetical-prediction path uses a session UUID instead — different code path.
- **Murmur3 collision**: 32-bit space → birthday-paradox collisions at ~65K elements. Acceptable for 1K buckets.
- **`traffic_pct = 0`**: every request goes to champion (no challenger traffic). Don't divide-by-zero.
- **`traffic_pct = 100`**: every A/B request goes to challenger. Allowed but unusual; document.
- **Floating-point pct**: `traffic_pct=33.33` → threshold `333.3` → rounded to 333. Document — accept ±1 bucket of rounding.
