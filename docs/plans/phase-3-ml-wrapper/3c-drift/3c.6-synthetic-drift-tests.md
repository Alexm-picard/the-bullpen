# 3c.6 — Synthetic drift tests

> Owning phase: `phase-3-ml-wrapper/3c-drift` · Estimated effort: `4–6 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:
- A test suite that proves each detector fires on a known-shifted distribution (decision `[64]`)
- Three categories of injected shifts:
  - Feature distribution: shift launch_speed mean by 1 std → PSI > 0.25
  - Prediction distribution: shift class proportions by 5pp → PSI > 0.1
  - Calibration: drop calibration error by 1.5× sustained 3 days → CALIBRATION_ERROR detector fires
- One CLI tool `python -m thebullpen.drift.fire_synthetic` for manual triggering during runbook drills

**OUT of scope**:
- Real-data drift detection — that's the production job in 3c.2–3c.5
- Alert wiring — 3c.7 consumes these tests as the truth source

---

## Objectives

1. All three injection types produce deterministic detector firings.
2. False-positive guard: zero injection on real data does NOT fire.
3. CLI tool produces a Discord NOTICE on demand for end-to-end alert testing.

## Step-by-step implementation tasks

1. Author Python `fire_synthetic.py` (CLI):
   ```bash
   uv run python -m thebullpen.drift.fire_synthetic --psi=0.4 --feature=count_balls
   uv run python -m thebullpen.drift.fire_synthetic --calibration-spike=1.6 --days=3
   uv run python -m thebullpen.drift.fire_synthetic --reset
   ```
   Inserts synthetic rows into `prediction_log` with the requested deviation; trigger a job manually; observe the metric.
2. Author `SyntheticDriftIT` (Java):
   - Inject 100 synthetic prediction rows where `count_balls` is shifted.
   - Run the `PsiFeatureJob` directly (call its `run()` method).
   - Query `drift_metrics` for the resulting row.
   - Assert `metric_value > 0.25`.
3. Repeat for prediction-shift and calibration-shift cases.
4. False-positive guard test: inject 100 rows from real distribution; assert metric < 0.05.

## Known edge cases

- **Synthetic data leaking into real metrics**: insert with a `synthetic=true` flag in the JSON `features` column; production drift queries should EXCLUDE these rows. Update 3c.2–3c.5 SQL to filter.
- **Reset path**: `--reset` deletes ALL rows where `features` JSON contains `synthetic=true`. Be careful — if the flag is missing, the reset doesn't clean up.
- **Test runtime**: each integration test pulls the entire job once. Cap to a small synthetic dataset (~1K rows).
- **Determinism**: seed RNG when injecting noise; same seed → same metric value.
