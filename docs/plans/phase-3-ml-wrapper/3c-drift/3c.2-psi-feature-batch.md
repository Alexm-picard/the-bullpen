# 3c.2 — Daily PSI on features

> Owning phase: `phase-3-ml-wrapper/3c-drift` · Estimated effort: `5–7 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:

- Daily Spring `@Scheduled` job (worker profile) computing PSI for each feature against the training-time distribution
- Continuous features: PSI with 10 quantile bins
- Categorical features: chi-squared distance (decision `[75]`)
- Reads recent predictions from `prediction_log` (the past 24h window)
- Reads training distribution from `metadata.json` (cached at startup)
- Writes one `drift_metric` row per (model_name, feature)

**OUT of scope**:

- Prediction-distribution PSI → 3c.3
- Calibration error → 3c.4
- Weekly segments → 3c.5
- Alerting → 3c.7

---

## Objectives

1. Daily job runs at ~2 AM ET (after games end), writes ~30 metric rows per active model.
2. PSI / chi² values match a hand-computed reference within `1e-6`.
3. Heartbeat to Healthchecks.io after success.

2. Author `TrainingDistributionLoader`:
   - On startup, for each active model_name, read `metadata.json` for the champion + (if shadow) challenger.
   - Cache reference distributions per (model_name, feature).
3. Author `PsiFeatureJob`:
   ```java
   @Scheduled(cron = "0 0 2 * * *", zone = "America/New_York")
   public void run() {
     for (String modelName : registry.activeModelNames()) {
       List<String> features = trainingDist.featuresFor(modelName);
       for (String feature : features) {
         double psi = clickhouse.query("SELECT ... PSI from prediction_log");
         repo.insertBatch(List.of(new DriftMetric(now(), modelName, championId, MetricType.PSI_FEATURE, feature, psi, ...)));
       }
     }
     healthchecks.ping("psi-feature-daily");
   }
   ```
   In practice, ClickHouse can compute the histograms for the past-24h window and the loader has the reference; the math runs in Java per pair.
4. Author `PsiTest`:
   - Reference and actual identical → PSI ≈ 0.
   - Mean shift of 1 std → PSI within expected range (~0.1).
   - Categorical: chi² zero on identical distributions.

## Known edge cases

- **Bin edges from reference**: if a feature is heavily skewed, quantile bins may collapse. Use equal-frequency bins on the reference, not equal-width.
- **Empty 24h window**: if a model had no traffic, write a row with `sample_size=0`. Don't crash.
- **`feature_hash` mismatch**: indicates mid-day model swap. Use only rows with matching hash for the active version.
- **Categorical with new value**: if `actual` has a category not in `reference`, treat reference count as 0 (smoothed); chi² still works.
