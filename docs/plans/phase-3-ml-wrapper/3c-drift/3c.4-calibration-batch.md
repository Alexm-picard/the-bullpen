# 3c.4 — Daily calibration on observed outcomes

> Owning phase: `phase-3-ml-wrapper/3c-drift` · Estimated effort: `4–6 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:
- Daily Spring `@Scheduled` job computing calibration metrics for each active model on yesterday's predictions joined with observed outcomes
- Metrics: Brier (multi-class), ECE, log loss
- Joins `prediction_log` with `pitches.description` (pitch model) or `pitches.events` (batted-ball model)
- Writes `drift_metrics` rows with `metric_type=BRIER` and `metric_type=CALIBRATION_ERROR`

**OUT of scope**:
- Per-segment calibration → 3c.5
- Alerting → 3c.7

---

## Objectives

1. Calibration metrics computed daily, written to `drift_metrics`.
2. Tracks the most-important drift signal per decision `[76]` (concept drift).
3. Joins handle late-arriving outcomes correctly (24h delay window).

## Step-by-step implementation tasks

1. Author `Calibration`:
   ```java
   public static double brier(List<float[]> proba, List<int[]> oneHotY) { ... }
   public static double ece(List<float[]> proba, List<int[]> oneHotY, int nBins) { ... }
   ```
2. Author `CalibrationJob`:
   ```java
   @Scheduled(cron = "0 30 2 * * *", zone = "America/New_York")
   public void run() {
     for (var model : registry.activeModels()) {
       var paired = clickhouse.query("""
         SELECT pl.prediction, p.description, p.events FROM prediction_log pl
         JOIN pitches p ON pl.feature_hash REGEX p.game_id  -- simplified
         WHERE pl.request_at BETWEEN now() - INTERVAL 48 HOUR AND now() - INTERVAL 24 HOUR
           AND pl.model_name = ? AND pl.role IN ('champion','shadow')""");
       double brier = Calibration.brier(...);
       double ece = Calibration.ece(...);
       repo.insertBatch(List.of(
         new DriftMetric(..., MetricType.BRIER, "all", brier, ...),
         new DriftMetric(..., MetricType.CALIBRATION_ERROR, "all", ece, ...)
       ));
     }
     healthchecks.ping("calibration-daily");
   }
   ```
3. Author `CalibrationTest`:
   - Perfectly calibrated synthetic predictions → ECE ≈ 0.
   - Brier on a Dirac-on-truth → 0.
   - Random predictions on uniform truth → Brier ≈ `(K-1)/K` for K classes.

## Known edge cases

- **Postponed games**: outcomes arrive much later. Re-run pulls them in; the same prediction may appear multiple times across days as outcomes settle. Use `request_at < now() - 24h` and accept that ~99% of outcomes are present at that point.
- **Suspended games / brief rain delays**: outcomes finalized once the game's `pitches` row exists. Use `pitches.game_pk` filter for completeness.
- **Truth source for batted-ball**: `pitches.events` is text; map to (out, 1B, 2B, 3B, HR) via a lookup. Document the mapping.
- **Non-paired predictions**: shadow rows must also be evaluated against truth. Use `role IN ('champion','shadow')`.
