## Known edge cases

- **Postponed games**: outcomes arrive much later. Re-run pulls them in; the same prediction may appear multiple times across days as outcomes settle. Use `request_at < now() - 24h` and accept that ~99% of outcomes are present at that point.
- **Suspended games / brief rain delays**: outcomes finalized once the game's `pitches` row exists. Use `pitches.game_pk` filter for completeness.
- **Truth source for batted-ball**: `pitches.events` is text; map to (out, 1B, 2B, 3B, HR) via a lookup. Document the mapping.
- **Non-paired predictions**: shadow rows must also be evaluated against truth. Use `role IN ('champion','shadow')`.
