# 3c.3 — Daily PSI on predictions

> Owning phase: `phase-3-ml-wrapper/3c-drift` · Estimated effort: `2–3 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:
- Same plumbing as 3c.2 but applied to predicted class probabilities (or argmax class distribution)
- Reference: training-time prediction distribution (from `metadata.json` or recomputed once on first startup)
- Output: `drift_metrics` rows with `metric_type=PSI_PREDICTION`, `feature_or_segment='<class_name>'` per class

**OUT of scope**:
- Calibration metrics → 3c.4

---

## Objectives

1. Daily PSI per class, per active model.
2. Sub-jobs run in the same `@Scheduled` cron as 3c.2 (or chained).

## Step-by-step implementation tasks

1. Author `PsiPredictionJob`:
   - For each active model, decode the past-24h `prediction` JSON from `prediction_log`.
   - Build per-class probability histogram (10 bins per class).
   - Compute PSI vs reference (training-time prediction distribution stored in `metadata.json` after Phase 2a.7).
   - Write metric rows.
2. Verify reference distribution exists in `metadata.json`. If not, modify 2a.7 to capture `training_prediction_distribution: {ball: [...], called_strike: [...], ...}` (10 quantile-bin frequencies per class) — author this back-edit as a small follow-up commit.
3. Tests:
   - Synthetic: same distribution → PSI ≈ 0.
   - Synthetic shift: 5% drop in `in_play` rate → PSI > 0.1.

## Known edge cases

- **Multi-class output**: 5-class for pitch, 5-class × 30-park for batted-ball. The batted-ball model writes 30 × 5 = 150 metric rows per day. Acceptable.
- **Reference distribution missing**: if a model was registered before 2a.7's metadata enhancement, fall back to recomputing reference from `prediction_log` over the first 7 days post-promotion.
- **Calibrated vs raw**: track on calibrated probabilities (the served output).
