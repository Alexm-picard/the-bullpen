## Step-by-step implementation tasks

1. Author `comparison.py`:
   ```python
   def compare(mlp_result: CVResult, lgbm_result: CVResult, test_data) -> ComparisonReport:
       mlp_per_park = per_park_metrics(mlp_predictions, test_data)
       lgbm_per_park = per_park_metrics(lgbm_predictions, test_data)
       # Side-by-side Brier, ECE, Confusion for each
       aggregate = {
         'mlp_brier': mlp_per_park.brier.mean(),
         'lgbm_brier': lgbm_per_park.brier.mean(),
         'mlp_ece': mlp_per_park.ece.mean(),
         'lgbm_ece': lgbm_per_park.ece.mean(),
         'better_brier': 'mlp' if mlp_per_park.brier.mean() < lgbm_per_park.brier.mean() else 'lgbm',
         'better_ece': 'mlp' if mlp_per_park.ece.mean() < lgbm_per_park.ece.mean() else 'lgbm',
       }
       return ComparisonReport(per_park=...,  aggregate=aggregate, ...)
   ```
2. Generate overlaid reliability diagrams (one per park, both curves on one figure).
3. Render HTML report at `ml/data/eval/batted_ball_comparison_v1.html` for human review.
4. Write `prefer_for_production` flag — chosen by Brier first, ECE as tiebreaker (defined upfront).
5. Author `test_comparison.py`: synthetic data → `compare` returns a populated report.

## Known edge cases

- **Tied metrics**: choose based on Brier; if tied, prefer the simpler model (LGBM).
- **Per-park inversion**: LGBM might win at park X but lose at park Y. Document — overall winner still chosen by mean.
- **Reliability diagram overlap visual**: 30 figures becomes a wall. Consider a small-multiples grid layout.
- **Comparison artifact ≠ eval artifact**: the per-model `eval/` directories already exist (from 2a.7). This artifact is the *comparison* on top.
