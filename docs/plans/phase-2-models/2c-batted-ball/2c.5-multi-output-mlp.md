# 2c.5 — Multi-output MLP (shared backbone + 30 per-park heads)

> Owning phase: `phase-2-models/2c-batted-ball` · Estimated effort: `12–18 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:

- PyTorch MLP: shared 2-layer backbone (Dense(64) → ReLU → Dense(64) → ReLU) + 30 parallel output heads, each Dense(5) for (out, 1B, 2B, 3B, HR)
- ~50K parameters total
- Inputs: 15 features (Statcast launch params + batter attrs + game state + atmosphere)
- Per-park softmax → 30 probability distributions per inference
- Training labels: retrodicted from 2c.4 (each BIP × 30 parks)
- ONNX export — 30-output multi-head topology
- Trained per fold via `cv_harness.run`; production model uses fold-4 split

**OUT of scope**:

- 30 isotonic calibrators → `2c.6`
- LightGBM Option-A baseline → `2c.8`
- Eval artifact comparison → `2c.9`

## Step-by-step implementation tasks

1. Add deps: `uv add torch onnx`.
2. Author `architecture.py`:
   ```python
   class BattedBallMLP(nn.Module):
       def __init__(self, n_features=15, n_parks=30, n_outcomes=5, hidden=64):
           super().__init__()
           self.backbone = nn.Sequential(
               nn.Linear(n_features, hidden), nn.ReLU(),
               nn.Linear(hidden, hidden), nn.ReLU(),
           )
           self.heads = nn.ModuleList([nn.Linear(hidden, n_outcomes) for _ in range(n_parks)])
       def forward(self, x):
           h = self.backbone(x)
           return torch.stack([self.heads[i](h) for i in range(len(self.heads))], dim=1)  # (B, 30, 5)
   ```
3. Author `dataset.py`:
   - `BBIPDataset(Dataset)`: yields `(features, labels_30x5)` pairs from joined `pitches` + `bbip_retrodicted_labels`.
   - Features: launch_speed, launch_angle, spray_angle, hit_distance, batter_handedness (1-hot 2 dims), sprint_speed, baserunners_state (1-hot 8 dims), outs, temp, wind_speed, wind_dir, humidity, altitude.
4. Author `train.py`:
   ```python
   def model_factory(train_loader, val_loader, seed=42, n_epochs=50):
       torch.manual_seed(seed)
       model = BattedBallMLP().cuda()
       optimizer = torch.optim.Adam(model.parameters(), lr=1e-3, weight_decay=1e-5)
       scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=n_epochs)
       loss_fn = nn.KLDivLoss(reduction='batchmean')  # since labels are distributions
       for epoch in range(n_epochs):
           model.train()
           for x, y in train_loader:
               logits = model(x.cuda()); log_probs = F.log_softmax(logits, dim=-1)
               loss = loss_fn(log_probs, y.cuda())
               loss.backward(); optimizer.step(); optimizer.zero_grad()
           scheduler.step()
           # eval on val_loader
       return model
   ```
5. Persist artifacts:
   - `model.pt` (PyTorch state dict)
   - `model.onnx` (`torch.onnx.export(model, dummy_input, opset_version=17)`)
   - `metadata.json`, `feature_pipeline.json`, `training_data.parquet`
6. Author `test_architecture.py`: forward pass with dummy `(B=8, 15)` returns `(8, 30, 5)`; per-park softmax sums to 1.
7. Author `test_train.py`: 1K-row synthetic dataset; loss decreases over 5 epochs.
8. Train production model:
   ```bash
   uv run python -m thebullpen.battedball.mlp.train --train 2015:2024 --val 2025 --version v1
   ```
9. Wire `cv_harness` for per-fold metrics; emit per-park test Brier and ECE in eval artifact.

## Known edge cases

- **GPU memory**: with 30 heads and batch 4096, model fits comfortably. Document peak ~2 GB GPU memory.
- **Determinism on GPU**: PyTorch on GPU is non-deterministic for some ops. Set `torch.use_deterministic_algorithms(True)` and `CUBLAS_WORKSPACE_CONFIG=:4096:8`. Document expected slight non-determinism if you can't enforce it fully.
- **KL divergence on Dirac labels**: when retrodicted distribution is degenerate (one outcome with prob 1), KL is infinite if the model assigns 0. Tiny smoothing (`labels = 0.99 * labels + 0.01/5`) fixes.
- **30 heads scaling**: park-relocation in v1.5 will require either a 31st head (with frozen prior heads) or full retrain. Document.
- **CI without GPU**: training tests must work CPU-only. Use `--device=auto` flag.
