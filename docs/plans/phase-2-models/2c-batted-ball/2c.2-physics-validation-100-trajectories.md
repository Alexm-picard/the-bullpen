# 2c.2 — Physics validation: 100 known trajectories ★ Phase 2c gate

> Owning phase: `phase-2-models/2c-batted-ball` · Estimated effort: `4–6 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:
- 100 hand-picked Statcast batted balls with known launch params + observed landing distance/hang-time
- Run each through the simulator and assert reconstructed landing distance is within ±5% of observed (or absolute ±15 ft for short flights)
- A clear "go/no-go" gate: if the validation fails, training cannot proceed (decision `[49]`)
- Validation report: `ml/data/physics_validation_report.json`

**OUT of scope**:
- Training labels → `2c.4`
- Park geometry → `2c.3`

**Hard rule**: This leaf must pass before any model in 2c trains. Decision `[49]` makes physics validation a precondition.

---

## Objectives

1. ≥ 95 of 100 trajectories within ±5% / ±15 ft of observed landing distance.
2. Mean absolute error on hang time ≤ 0.3 seconds.
3. Validation report committed; failure gates training.
4. Disagreements documented (some Statcast rows have known measurement issues — flag, don't quietly skip).

## Step-by-step implementation tasks

1. Curate 100 trajectories:
   - 50 home runs (full carry distance well-measured),
   - 30 line drives / fly outs (mid-distance),
   - 20 short pop-ups / weak grounders.
   - Spread across parks, weather conditions, batter handedness.
   - Pull from `pitches` joined with weather pulls (when available) or use stadium-default atmosphere.
   - Each fixture: `launch_speed, launch_angle, spray_angle, spin_rate, spin_axis, atmosphere, observed_distance_ft, observed_hang_time_s`.
2. Author `validate.py`:
   ```python
   def run_validation(fixtures_path: Path) -> dict:
       fixtures = json.loads(fixtures_path.read_text())
       results = []
       for row in fixtures:
           traj = simulate(LaunchParams(**row['launch']), Atmosphere(**row['atmo']))
           pred_distance_ft = traj.distance_m * 3.28084
           pred_hang_time = traj.hang_time
           err_distance = abs(pred_distance_ft - row['observed_distance_ft']) / row['observed_distance_ft']
           err_hang = abs(pred_hang_time - row['observed_hang_time_s'])
           results.append({**row, 'pred_distance_ft': pred_distance_ft, 'pred_hang_time': pred_hang_time,
                           'err_distance_pct': err_distance, 'err_hang_s': err_hang,
                           'pass_distance': err_distance <= 0.05 or abs(pred_distance_ft - row['observed_distance_ft']) <= 15,
                           'pass_hang': err_hang <= 0.5})
       return {
           'n_fixtures': len(fixtures),
           'pass_rate_distance': sum(r['pass_distance'] for r in results) / len(fixtures),
           'mae_hang_s': sum(r['err_hang_s'] for r in results) / len(fixtures),
           'failures': [r for r in results if not r['pass_distance']],
       }
   ```
3. Run: `uv run python -m thebullpen.battedball.physics.validate`. Write report to `ml/data/physics_validation_report.json`.
4. Inspect failures. If pass rate < 95%, debug:
   - Are spin rate units right?
   - Is the atmosphere correct for that game's conditions?
   - Is `spray_angle` measured the same way Nathan defined it?
   Fix; re-validate. **Do not proceed to 2c.3+ until pass rate ≥ 95%.**
5. Author `test_validate.py`: load 5 of the 100 fixtures (tiny subset for CI speed); assert pass rate = 100% on the subset.
6. Add a CI guard: if `physics_validation_report.json` shows pass rate < 95%, fail CI (gates merges to main).

## Known edge cases

- **Statcast inaccuracy on weak hits**: pop-ups and grounders have noisy distance values. Tolerance widened to absolute ±15 ft to cover.
- **Wind during HR validations**: wind can shift HR carry by 30+ ft. Pick fixtures from low-wind games OR use observed wind from weather archive.
- **Park-specific air**: Coors and altitude parks need true atmospheric pressure. Use park-specific defaults for fixtures not from real games.
- **Spin axis 12-clock convention**: Statcast uses clock face. Convert to physical tilt angle inside the fixture loader.
- **Failure surface**: if simulator fails the gate, do NOT switch to soft-cut (Path B) without a `decisions.md` reversal entry.
