# 2c.1 — Physics simulator (drag/Magnus ODE, RK4)

> Owning phase: `phase-2-models/2c-batted-ball` · Estimated effort: `12–16 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:

- Python ball-flight simulator (~200 lines) implementing Alan Nathan's published drag + Magnus equations
- Equations of motion: gravity, drag (CD ≈ 0.34, with v-dependent correction), Magnus lift (lift coefficient as a function of spin parameter S = ωr/v)
- RK4 numerical integrator with adaptive step size
- Inputs: launch_speed, launch_angle, spray_angle, spin_rate, spin_axis, plus atmosphere (temp, RH, pressure, altitude, wind vector)
- Outputs: full trajectory (time series of x, y, z, vx, vy, vz), plus landing point, max height, hang time
- Vectorized for batches (N trajectories at a time) for performance

**OUT of scope**:

- Validation against 100 trajectories → `2c.2-physics-validation-100-trajectories.md`
- Park geometry (where the ball lands relative to walls/seats) → `2c.3-park-geometry-data.md`
- Retrodiction labeling pipeline → `2c.4-retrodiction-labeling-pipeline.md`

---

## Objectives

1. `simulate(launch_params, atmosphere, n_steps_max=2000) -> Trajectory` runs in <1 ms per trajectory on CPU.
2. Vectorized `simulate_batch(...)` runs 10K trajectories/sec.
3. Code is unit-tested against analytical expectations (vacuum projectile motion at zero spin, zero drag).
4. Public API is small: 1 `Trajectory` dataclass + 2 functions (`simulate`, `simulate_batch`).

## Step-by-step implementation tasks

1. Author `_constants.py`:
   - `BALL_MASS_KG = 0.145`, `BALL_RADIUS_M = 0.037`, `g = 9.80665 m/s²`.
   - Drag/lift coefficient lookup tables from Nathan's published data.
2. Author `atmosphere.py`:
   - `air_density(temp_C, pressure_hPa, altitude_m, humidity_pct) -> kg/m³` using ideal gas law + humidity correction.
3. Author `equations.py`:
   - `drag_force(velocity_vec, air_density) -> ndarray` — CD scaled to current Reynolds number.
   - `magnus_force(velocity_vec, spin_rate_rad_s, spin_axis_unit, air_density) -> ndarray`.
   - `total_acceleration(state, atmosphere) -> ndarray` — sum of all forces / mass + gravity.
4. Author `integrator.py`:
   - RK4 step: `state_{n+1} = rk4_step(state_n, dt, accel_fn)`.
   - Adaptive timestep: smaller near apex, larger during cruise.
5. Author `simulator.py`:

   ```python
   @dataclass
   class Trajectory:
       t: np.ndarray  # (N,) seconds
       pos: np.ndarray  # (N, 3) meters
       vel: np.ndarray  # (N, 3) m/s
       landed: bool
       hang_time: float
       distance_m: float
       max_height_m: float

   def simulate(launch: LaunchParams, atmo: Atmosphere) -> Trajectory: ...
   def simulate_batch(launches: list[LaunchParams], atmos: list[Atmosphere]) -> list[Trajectory]: ...
   ```

   Vectorize the batch version using NumPy broadcasting (state matrix shape `(N, 6)`).

6. Author `test_equations.py`:
   - At zero spin: `magnus_force` returns zero vector.
   - At zero velocity: `drag_force` returns zero vector.
   - At sea-level standard atmosphere: `air_density ≈ 1.225 kg/m³` within 1%.
7. Author `test_simulator.py`:
   - **Vacuum check**: with `air_density=0` and `spin_rate=0`, simulator matches analytical projectile motion within `1e-3` over 4-second flight.
   - **Sanity**: 100 mph / 28° / no spin / standard air lands ~330–400 ft (well-known typical value).
   - **Determinism**: same inputs → byte-identical trajectory.

## Known edge cases

- **Spin-rate units**: rpm vs rad/s. Document choice (`rpm` external, `rad/s` internal).
- **Spin axis convention**: tilt angle from horizontal. Use Nathan's convention.
- **Numerical stability near landing**: detect `pos.z <= 0` and interpolate to find exact landing point.
- **Wind vector**: 3D in stadium frame; convert from "direction wind is FROM" (degrees) to vector pointing TOWARDS where wind goes.
- **Reynolds-number correction**: CD jumps near 80 mph. Lookup-table interpolation; document discontinuity.
- **Python perf**: pure-Python loops are too slow. Use NumPy vectorization OR Numba JIT for the RK4 loop. Prefer NumPy first; revisit if profiling demands Numba.
