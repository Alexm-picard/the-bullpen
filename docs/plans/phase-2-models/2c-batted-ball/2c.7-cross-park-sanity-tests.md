## Scope boundaries

**IN scope**:

- A test suite asserting the MLP's 30-output predictions obey **monotonicity** with respect to known park HR-friendliness rankings
- Specifically: for a canonical input (e.g., 110 mph / 28┬░ launch / no wind / median atmospheric conditions), the predicted P(HR) at each park should rank approximately the same as published park HR factors
- Spearman rank correlation > 0.8 between predicted and published HR factors
- Spot-check: predicted P(HR) at COL > P(HR) at OAK (large gap), and the gap is not absurd

**OUT of scope**:

- A separate Java port of these tests (not needed; Python is enough)
- Live-game predictions

**Hard rule** (decision `[52]`): if cross-park sanity fails, the model is broken; do not register it.

---

## Objectives

1. Sanity test passes for the production `batted_ball/v1` model.
2. Failure produces an actionable error message ("park X predicts P(HR)=0.05 but its published HR factor is 1.4 ÔÇö likely overfitting").
3. CI runs the test as a fast integration check.

## Known edge cases

- **Published HR factors vary by source**: pick one source per release of the data. Document in JSON. Don't mix.
- **Year-over-year fluctuations**: HR factors change year-to-year. Use the year matching the model's validation year.
- **Domes / retractable roofs**: published factors include weather effects we exclude in canonical input. Some leniency in the rank correlation is expected (hence threshold 0.8 not 0.95).
- **Failure mode**: if model fails sanity, do not coerce data; investigate. Likely culprits: bad retrodiction labels, MLP underfit, wrong canonical input encoding.
