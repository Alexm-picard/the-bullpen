## Step-by-step implementation tasks

1. Flatten the retrodicted labels: each `(bbip_id, park_id)` row becomes a training example with `park_id` as a categorical feature. Label = argmax of the retrodicted distribution (or sample from it).
2. Author `train.py`:
   ```python
   FEATURES = [..., "park_id"]
   def model_factory(train, val, seed=42):
       params = dict(objective="multiclass", num_class=5, metric="multi_logloss",
                     learning_rate=0.05, num_leaves=63, seed=seed,
                     deterministic=True, force_row_wise=True)
       dtrain = lgb.Dataset(train[FEATURES], label=train["label"], categorical_feature=["park_id"])
       booster = lgb.train(params, dtrain, num_boost_round=2000, valid_sets=[dtrain, lgb.Dataset(val[FEATURES], label=val["label"])], callbacks=[lgb.early_stopping(50)])
       return ModelBundle(booster, IsotonicCalibrator.fit(val.label, booster.predict(val[FEATURES])))
   ```
3. Persist to `/var/lib/thebullpen/models/batted_ball_lgbm_baseline/v1/` with full eval artifact.
4. Run `cv_harness.run(...)` and `eval/artifact.generate(...)`.

## Known edge cases

- **Serving 30 inferences**: not a concern in v1 (we don't serve this baseline). If we ever do, batch the 30 per-park calls into a single LGBM call by stacking the rows.
- **Labels via argmax of distribution**: loses information. Document. An alternative — sample from distribution per training example — averages the same way but adds variance. Pick argmax for v1 simplicity.
- **`park_id` cardinality**: 30 — LightGBM handles natively.
- **Memory**: 30× expansion of the BIP table → ~100M training rows. LightGBM hist-based trains fine; if memory tight, downsample non-home-park rows.
