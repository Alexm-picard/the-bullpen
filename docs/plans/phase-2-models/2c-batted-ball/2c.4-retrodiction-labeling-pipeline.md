# 2c.4 — Retrodiction labeling pipeline

> Owning phase: `phase-2-models/2c-batted-ball` · Estimated effort: `8–12 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:

- For every BIP in `pitches`, run the simulator 30 times (once per park) using the BIP's launch parameters
- Each park's atmosphere comes from the post-game weather observed pull (or stadium default if no weather)
- Output: `bbip_retrodicted_labels` table — one row per (BIP, park) — 30 rows per BIP
- Outcome distribution per (BIP, park): probability vector over (out, 1B, 2B, 3B, HR), derived from a small Monte Carlo around the deterministic simulation (e.g., 10 simulator runs with small Gaussian noise on launch params + light fielder-positioning randomness)

**OUT of scope**:

- The MLP that consumes these labels → `2c.5`
- 30 isotonic calibrators → `2c.6`
- Eval artifact comparing MLP vs LGBM → `2c.9`

---

## Objectives

1. Pipeline produces ~3 M effective training examples per park (every BIP × every park).
2. Retrodicted labels for the home park match observed outcomes within a noise budget (validation: HR rate at home park within ±2% of observed HR rate).
3. Run completes ≤ 24 hours wall time on the workstation (parallel via `multiprocessing`).

## Step-by-step implementation tasks

1. V010:
   ```sql
   CREATE TABLE bbip_retrodicted_labels (
     bbip_id String,                  -- (game_id || at_bat_index || pitch_number)
     park_id LowCardinality(String),
     prob_out Float32, prob_1b Float32, prob_2b Float32, prob_3b Float32, prob_hr Float32,
     observed_outcome Enum8('out'=0,'1b'=1,'2b'=2,'3b'=3,'hr'=4) NULL,  -- NULL except for the home park
     ingested_at DateTime DEFAULT now()
   ) ENGINE = ReplacingMergeTree(ingested_at)
   PARTITION BY toYYYYMM(toDate(substring(bbip_id, 1, 8)))
   ORDER BY (park_id, bbip_id);
   ```
2. Author `_atmospheres.py`: for every (game_id, park_id) needed, return `Atmosphere`. Pull from `weather_observed` (if available) for home park; for non-home parks, use that park's default atmosphere (since we're retrodicting "what would happen if this BIP were hit at park X under typical conditions").
3. Author `labels.py`:
   ```python
   def retrodict_one(bbip: BBIP, park: ParkGeometry, atmo: Atmosphere, n_mc: int = 10, seed: int = 42) -> dict:
       outcomes = []
       rng = np.random.default_rng(seed + hash(bbip.id) % 2**32)
       for _ in range(n_mc):
           jitter = LaunchParams(
             launch_speed = bbip.launch_speed + rng.normal(0, 0.5),
             launch_angle = bbip.launch_angle + rng.normal(0, 0.7),
             spray_angle = bbip.spray_angle + rng.normal(0, 0.5),
             spin_rate = bbip.spin_rate, spin_axis = bbip.spin_axis
           )
           traj = simulate(jitter, atmo)
           outcomes.append(classify_outcome(traj, park, league_mean_speed_fps=27.0))
       counts = Counter(outcomes)
       total = sum(counts.values())
       return {f"prob_{o.lower()}": counts.get(Outcome[o.upper()], 0) / total
               for o in ['out','1b','2b','3b','hr']}
   ```
4. Author `run_pipeline.py`:
   - Stream BIPs from `pitches FINAL WHERE description='in_play'` in chunks of 1000.
   - For each BIP: load its atmosphere, then run `retrodict_one` for all 30 parks (using `multiprocessing.Pool`).
   - Insert results into `bbip_retrodicted_labels`.
   - Idempotency: drop partition for the BIP's month before re-load OR check existence.
5. Author `test_labels.py`: small synthetic BIP + 2 parks → assert outcome probabilities sum to 1, are in expected ranges (high-velocity / high-angle ball at NYA ≈ high HR prob).
6. Run: full pipeline on 2015–2024 data. Validate: HR rate predicted at home park (across all BIPs) should match observed HR rate within ±2%.

## Full-run procedure (desktop)

Run on the desktop per ADR-0006 (dev on Mac, prod on desktop):

```bash
# Single season (smoke test)
uv run python -m bullpen_training.battedball.retrodict.run_pipeline \
    --season 2024 --report training/data/retrodict_report_2024.json

# Full 2015-2024 backfill (overnight)
uv run python -m bullpen_training.battedball.retrodict.run_pipeline \
    --season-from 2015 --season-to 2024 \
    --report training/data/retrodict_report_full.json 2>&1 | tee logs/retrodict_full.log
```

`ReplacingMergeTree` dedupes by natural key on merge, so the pipeline
can be safely re-run; the latest `ingested_at` wins.

## Known edge cases

- **`spray_angle` definition**: Statcast's `hc_x/hc_y` need conversion. Document the formula in `bbip_id` extraction.
- **Foul balls**: BIPs that landed foul → outcome 'foul' not in our 5-class. Skip foul BIPs entirely.
- **Out + advanced runner**: simplification — outcome is the batter's outcome only; runners aren't modeled in v1.
- **No-spin / unknown spin**: if `spin_rate IS NULL`, use league-mean spin for the pitch type.
- **Home-park atmosphere fallback**: if `weather_observed` row is missing for a game, use park default. Track count of fallback rows; if >5%, surface a warning.
