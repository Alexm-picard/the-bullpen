> Owning phase: `phase-2-models/2b-pitch-post` · Estimated effort: `4–5 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:
- Add Tier 4 columns to the `features` table (additive migration):
  - `pitch_type` (already in `pitches`; surfaced into `features`)
  - `release_speed_mph`, `plate_x_in`, `plate_z_in` (already in `pitches`)
  - Derived: `release_pos_x`, `release_pos_z`, `pfx_x`, `pfx_z` (movement) — pull from `raw_statcast` since not yet in `pitches`
  - `spin_rate`, `spin_axis` (also from raw)
- Update `pitches` cleaning step (1.2) to surface these columns OR add a new V008 migration that adds them post-hoc

**OUT of scope**:
- Sequence features (previous N pitches) → v1.5 deferred
- Catcher / umpire → v1.5 deferred (decision `[41]`)

3. Pipeline run takes ≤ 1 hour.

## Step-by-step implementation tasks

1. V008 — `ALTER TABLE pitches ADD COLUMN pfx_x_in Nullable(Float32), pfx_z_in Nullable(Float32), spin_rate_rpm Nullable(Float32), spin_axis_deg Nullable(Float32), release_pos_x_in Nullable(Float32), release_pos_z_in Nullable(Float32);`
2. Update `transform_raw_to_pitches.sql` to populate the new columns from `raw_statcast`.
3. Re-run `transform_pitches.py` for all years to backfill.
4. V009 — `ALTER TABLE features ADD COLUMN pitch_type LowCardinality(String), release_speed_mph Nullable(Float32), plate_x_in Nullable(Float32), plate_z_in Nullable(Float32), pfx_x_in Nullable(Float32), pfx_z_in Nullable(Float32), spin_rate_rpm Nullable(Float32), spin_axis_deg Nullable(Float32), release_pos_x_in Nullable(Float32), release_pos_z_in Nullable(Float32);`
5. Author `tier_4_postpitch.py`:
   - `build_features_t4(client, fold_id)`: pulls from `pitches` and updates `features` with Tier 4 columns for pitches in fold.
6. Run for all 4 folds.
7. Author `test_tier_4.py`: round-trip 100 rows; assert non-null Tier 4 values for in-play pitches.

---

## Known edge cases

- **Missing tracking data**: ~3-5% of historical pitches have `release_speed=NULL`. Model must accept Nullable inputs (LightGBM handles natively).
- **`pitch_type='UN'`**: ~1% of pitches. Treat as a valid category; don't drop rows.
- **Tier 4 used in `pre` model**: catastrophic leakage. The `feature_pipeline.json` for `pitch_outcome_pre` MUST NOT list Tier 4 columns. Enforced by the leakage tests.
