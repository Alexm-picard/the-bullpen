# 2b.2 — Train + eval post-pitch model

> Owning phase: `phase-2-models/2b-pitch-post` · Estimated effort: `5–7 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:

- Reuse 2a.5's training code with a different feature set (Tiers 1+2+3+**4**)
- New model_name: `pitch_outcome_post`
- Independent CV run, independent calibrator, independent eval artifact
- LightGBM only — no separate LR baseline for the post head (LR baseline is `pitch_outcome_pre`-only by design)

**OUT of scope**:

- Serving the post head endpoint → 2b.3
- ONNX export → 2b.3

---

## Objectives

1. `pitch_outcome_post/v1` registered with full eval artifact.
2. Post-head Brier < pre-head Brier on the same test years (post-pitch features add real signal).
3. ECE post-calibration < 0.02 on test years.

---

## Dependencies

**Upstream**: `2a.4` (CV harness), `2a.5` (training code), `2a.7` (artifact generator), `2b.1` (Tier 4 features).

---

## Required files / modules

**New**:

- `ml/thebullpen/pitch/train_post.py` (mostly imports `train_pre.py` with a different FEATURES list)
- `ml/tests/pitch/test_train_post.py`

---

## Step-by-step implementation tasks

1. Author `train_post.py`:
   ```python
   FEATURES_POST = FEATURES_PRE + [
       "pitch_type",          # one-hot or LightGBM categorical
       "release_speed_mph", "plate_x_in", "plate_z_in",
       "pfx_x_in", "pfx_z_in",
       "spin_rate_rpm", "spin_axis_deg",
       "release_pos_x_in", "release_pos_z_in",
   ]
   def model_factory(train, val, seed=42):
       # Same code path as 2a.5 but with FEATURES_POST and model_name='pitch_outcome_post'
       ...
   ```
2. Run via `cv_harness.run(...)`.
3. Persist to `/var/lib/thebullpen/models/pitch_outcome_post/v1/` with all artifacts.
4. Run `eval/artifact.generate(...)`.
5. Smoke test: post-head test Brier < pre-head test Brier (same fold).

---

## Testing requirements

**Unit**: feature list separation (no Tier 4 leaks into pre).
**Integration**: 5K-row synthetic train; ECE drops post-calibration.
**CI**: `test-python.yml`.

---

## Acceptance criteria

- [ ] `pitch_outcome_post/v1` artifacts present.
- [ ] Post-head Brier < pre-head Brier on every fold's test year.
- [ ] ECE post-calibration < 0.02 on test years.
- [ ] `feature_pipeline.json` lists all Tier 4 columns; `feature_schema_hash` differs from `pitch_outcome_pre`.

---

## Known edge cases

- **Pitch-type as categorical**: LightGBM handles natively if dtype is `category` in pandas. Document the dtype expectation.
- **Missing tracking**: ~5% of pitches have NULLs in Tier 4. LightGBM handles missing natively. Accept.
- **Same `feature_schema_hash` accident**: include `model_name` in the hash to make pre and post distinct even if columns somehow matched.

| 2026-05-24 | **Hang root cause fixed; live production bundle persisted via `--skip-cv`.** Resumed in a fresh session focused on debugging. The "hang" was **not** a hang — it was **stdout buffering**. LightGBM 4.6.0's default `_log_info` callback calls bare `print(msg)` to `sys.stdout`. When the CLI is invoked under `... 2>&1 \| tee log`, Python defaults to **block-buffered** stdout (~8 KB), so 100 iteration log lines × ~50 chars = 5 KB sits in the buffer the entire 2000-round run. Training was completing fine; the user saw zero `[100]` log lines because they were never flushed. py-spy confirmed CPU was actually grinding in `lgb.train` C++ code throughout the apparent "hang". **Fix:** route LightGBM's logger through Python's `logging` module (line-buffered via `StreamHandler.emit()` which `flush()`es every record). One-line change in `production.py`'s `main()` after `configure_logging()`: `lgb.register_logger(logging.getLogger("lightgbm"))`. **Second fix (latent crash):** `persist.py::persist_lightgbm_v1` hardcoded `PITCH_FEATURE_COLUMNS` (31 cols) when calling `lightgbm_feature_importance(bundle.booster, …)`; on a post-head bundle that returns 41 importance values, `pd.DataFrame({feature: 31-len, importance_gain: 41-len})` would have raised `ValueError: arrays must all be same length` at persist time. Switched to `list(bundle.feature_cols)` so the pre head still gets 31 and the post head gets 41. **Verification (production --skip-cv, fold 4):** complete end-to-end run with default `num_boost_round=2000` in **13:01 wall** — all 4 CV folds skipped, fold-4 production training stopped at round 1300 by `early_stopping(50)`, val multi_logloss = 0.97276, predict_proba on test_2025, calibrator fit, full bundle persisted at `training/artifacts/pitch_outcome_post/v1/` (model.lgb 45 MB, calibrator.json, metadata.json, feature_pipeline.json, park_id_mapping.json, pitcher_te.json, batter_te.json, training_data.parquet 24 MB, eval/ with metrics.json + reliability_diagrams.png + confusion_matrix.png + feature_importance.csv + segment_metrics.csv + temporal_cv_results.csv). **Partial-CV evidence (full run interrupted at 30 min in fold 4):** since folds 1-3 are independent, ran them through `--model post` (no `--skip-cv`) to get acceptance metrics.
