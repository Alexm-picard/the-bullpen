# 2a.1 — Feature pipeline (Tiers 1 + 2)

> Owning phase: `phase-2-models/2a-pitch-pre` · Estimated effort: `8–12 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:

- Tier 1 (trivially safe): count, outs, inning, base_state, score_diff, pitcher×batter handedness, park_id, day-of-week from game_date
- Tier 2 (identity, target-encoded): pitcher_id, batter_id via leakage-safe rolling target encoding with strict pre-game cutoff
- Out-of-fold encoding: per-fold target-encoding lookup files
- ClickHouse `features` table with the Tier 1+2 columns + `pitch_pk` and `as_of_date`
- Python module `ml/thebullpen/features/` exporting `build_features_t12(client, year)` and `make_target_encodings(...)` callable per fold

**OUT of scope**:

- Tier 3 (rolling form features) → `2a.2-feature-pipeline-tier-3-form.md`
- Tier 4 (post-pitch features) → Phase 2b
- Weather features → deferred to Phase 2 weather-pull leaf
- Java-side feature pipeline reproduction → 2a.8

---

## Objectives

1. `features` table populated for all 5-class label-eligible pitches (`description != 'unknown'`) for 2015–2025.
2. Target encoding strictly uses data with `game_date < pitch_date` (leakage-safe by construction).
3. Each fold produces its own `target_encoding_<entity>_fold<k>.json` lookup file under `ml/data/encodings/`.
4. Re-running is idempotent on a per-fold basis.

## Step-by-step implementation tasks

1. Backfill 2015–2023 by re-running `statcast_pull.py --season <Y>` for each year. Allocate one weekend (Risk Register G12 — pattern proven in 1.1).
2. Re-run `transform_pitches.py --year <Y>` for each new year so `pitches` has the full 2015–2025 corpus.
3. Author `V005__features.sql`:
   ```sql
   CREATE TABLE IF NOT EXISTS features (
     game_id UInt64, at_bat_index UInt16, pitch_number UInt8,
     game_date Date, as_of_date Date,
     pitcher_id UInt32, batter_id UInt32,
     -- Tier 1
     count_balls UInt8, count_strikes UInt8, outs UInt8, inning UInt8,
     base_state UInt8, score_diff Int16,
     pitcher_throws FixedString(1), batter_stand FixedString(1),
     park_id LowCardinality(String), dow UInt8,
     -- Tier 2
     pitcher_te_ball Float32, pitcher_te_strike Float32, pitcher_te_inplay Float32,
     batter_te_ball Float32, batter_te_strike Float32, batter_te_inplay Float32,
     fold UInt8,
     ingested_at DateTime DEFAULT now()
   ) ENGINE = ReplacingMergeTree(ingested_at)
   PARTITION BY (toYYYYMM(game_date), fold)
   ORDER BY (fold, game_date, game_id, at_bat_index, pitch_number);
   ```
4. Author `target_encoding.py`:
   - Function `compute_te(client, entity_col, label_col, train_window) -> DataFrame[entity, te_*]` doing class-conditional smoothed encoding (Bayesian smoothing prior = global rate).
   - Function `apply_te(df, encoding_df, entity_col)` doing left-join with smoothed default for unseen entities.
   - Strict cutoff: every encoding lookup is built from `pitches WHERE game_date < <fold_test_window_start>`.
5. Author `tier_1_2.py`:
   - `build_features_t12(client, fold_id, train_dates, test_dates)`:
     1. Build pitcher and batter TE from `train_dates`.
     2. Persist encodings to `ml/data/encodings/<entity>_fold<k>.json`.
     3. Compute Tier 1 columns directly via SQL (`count_balls`, etc.).
     4. Insert rows into `features` with `fold=<k>`.
6. Author `_sql.py`/`build_features_t12.sql` materializing Tier 1 columns from `pitches` joined with the persisted TE files (read into a temp table or applied client-side via pandas merge).
7. Author tests:
   - `test_target_encoding.py`: tiny synthetic dataset, assert encoded values match hand-computed values; assert prior smoothing applied for unseen IDs.
   - `test_tier_1_2.py`: round-trip 100 rows from a fixture `pitches`-like table → `features`; assert Tier 1 columns match expected.
8. Run for all 4 folds against the full corpus; verify `SELECT count(*) FROM features WHERE fold=k` is non-zero per fold.

## Testing requirements

**Unit**:

- TE smoothing for low-N entities matches expected formula.
- TE applied to unseen ID returns global prior.
- `base_state` bitmask matches `pitches.base_state` byte-for-byte.

**Integration**:

- Round-trip end-to-end on a 100-row fixture; counts and column dtypes match.

**Leakage** (preview of 2a.3):

- Manual: pick 1 random pitch, walk back through TE lookup, confirm only `game_date < pitch_date` rows contributed.

**CI gates**: `test-python.yml`.

## Known edge cases

- **Cold-start pitchers / batters**: TE returns global prior. Document that downstream models should treat the "global prior" value as a soft "unknown" signal (not zero).
- **Mid-season debutants**: their TE per pitch is built only from prior pitches; very early pitches use prior. Acceptable.
- **`score_diff` sign convention**: home minus away regardless of who's batting; encode handedness elsewhere.
- **`base_state` overlap**: bitmask values 0..7 encode all 8 combinations cleanly.
- **`fold` partition explosion**: 4 folds × 12 months × ~10 years ≈ 480 partitions. ClickHouse handles this fine; just don't add more dimensions to PARTITION BY.
