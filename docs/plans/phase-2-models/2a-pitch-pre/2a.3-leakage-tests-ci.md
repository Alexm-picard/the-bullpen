## Step-by-step implementation tasks

1. Build the fixture: `uv run python -m thebullpen.features.build_features_t12 --fold 1` then dump `features` rows for a 30-day window to `leakage_sample.parquet`.
2. `test_no_future_contamination.py`:
   - Pick a cutoff date `T`. Corrupt all `pitches` rows with `game_date >= T` to set `description='ball'`.
   - Re-run `build_features_t12` for fold 1 (test window after `T`).
   - Train a tiny LightGBM, evaluate on a held-out test from before `T`.
   - **Assertion**: test Brier should NOT regress more than 5% — corruption AFTER cutoff must not affect features used BEFORE cutoff.
3. `test_shuffled_target.py`:
   - Shuffle the `description` label column.
   - Train tiny LightGBM, eval test Brier.
   - **Assertion**: test Brier ≈ random-guess floor (within tolerance: ≤ 0.005 of `1 - 1/n_classes` Brier baseline).
4. `test_calendar_date_trace.py`:
   - Pick 10 random pitches.
   - For each: walk every Tier 1 / 2 / 3 value via SQL queries, asserting source data has `game_date < pitch_date` (Tier 2 TE) or `as_of_date < pitch_date` (Tier 3 windows).
5. `test_id_consistency.py`:
   - Construct two synthetic pitches with identical `(pitcher_id, game_date_history)` but different positions in the dataset.
   - Run feature pipeline; assert TE values are identical to within `1e-9`.
6. Update CI workflow:
   ```yaml
   - name: Leakage tests
     working-directory: ml
     run: uv run pytest ml/tests/leakage/ -v --strict-markers
   ```
   Add a hook that fails CI if any leakage test has `@pytest.mark.skip` without an `# ADR: decisions.md#NN` comment on the same line.
7. Mutation-test the suite once:
   - Temporarily allow future contamination (remove cutoff filter). Confirm `test_no_future_contamination` FAILS.
   - Temporarily remove the shuffled-target safeguard (e.g., a hidden leak via `pitcher_id` alone). Confirm `test_shuffled_target` FAILS.
   - Revert mutations. Document the mutation runs in the PR description.

## Known edge cases

- **Test runtime**: `test_no_future_contamination` rebuilds features — slow. Cap to fold 1 only on the sample fixture.
- **`test_shuffled_target` flakiness**: Brier baseline depends on class proportions. Compute baseline empirically per run, not as a constant.
- **`test_calendar_date_trace` strictness**: `<` vs `<=`. Pre-game cutoff is strict less-than. Encode in fixture: a pitch on `2024-04-15` must not see any data dated `2024-04-15`.
- **Skip-without-ADR detector**: the simplest implementation is a grep for `@pytest.mark.skip` outside of `# ADR:` lines. If you ever need an unconditional skip, add the ADR.
