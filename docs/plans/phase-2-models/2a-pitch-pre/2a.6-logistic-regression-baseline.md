## Scope boundaries

**IN scope**:

- Multinomial logistic regression on Tier 1+2+3 features (decision `[37]`)
- Same isotonic calibration step as the LightGBM model (consistent comparison)
- Registered as `pitch_outcome_lr_baseline` (not `pitch_outcome_pre`) — different model_name on purpose

- Hyperparameter search (use sklearn defaults + L2 regularization + standard scaler)

## Objectives

1. LR baseline trained on the same fold splits as LightGBM and registered with full eval artifact.
2. LR Brier > LightGBM Brier (LR loses) — establishes the floor (decision `[37]`).
3. LR is the smoke test: if LightGBM ever underperforms LR, something is wrong with the LightGBM training.

## Step-by-step implementation tasks

1. Author `train_lr_baseline.py`:

   ```python
   from sklearn.linear_model import LogisticRegression
   from sklearn.preprocessing import StandardScaler
   from sklearn.pipeline import Pipeline

   def model_factory(train_df, val_df, seed=42):
       pipe = Pipeline([
         ("scaler", StandardScaler()),
         ("lr", LogisticRegression(multi_class="multinomial", solver="lbfgs",
                                   max_iter=1000, C=1.0, random_state=seed))
       ])
       pipe.fit(train_df[FEATURES], train_df["label"])
       val_pred = pipe.predict_proba(val_df[FEATURES])
       calibrator = IsotonicCalibrator.fit(val_df["label"], val_pred)
       return ModelBundle(pipe, calibrator)
   ```

2. Run via `cv_harness.run(model_factory, feature_loader, [brier, log_loss, ece])`.
3. Persist to `/var/lib/thebullpen/models/pitch_outcome_lr_baseline/v1/`. **Note** `model.lgb` is not relevant here — store `model.pkl` (joblib) AND ONNX export via skl2onnx.
4. Author `test_train_lr.py`: 1K-row synthetic dataset, train, assert non-trivial accuracy (>uniform).

## Known edge cases

- **Categorical feature handling**: LR can't natively use categoricals. We rely on TE having pre-encoded them; one-hot for `pitcher_throws` / `batter_stand` / `park_id` (LowCardinality OK) inside the Pipeline.
- **`max_iter` warnings**: bump to 2000 if convergence warnings appear.
- **ONNX export of sklearn Pipeline**: use `skl2onnx.convert_sklearn(pipe, initial_types=...)`. Numeric features only; the StandardScaler step embeds in the ONNX graph (Risk Register G4 alignment).
- **Permanent registration**: when 3a.3 enforces feature-schema hashes, the LR baseline must share the production feature pipeline — re-train if the pipeline ever changes.
