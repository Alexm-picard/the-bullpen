> Owning phase: `phase-2-models/2a-pitch-pre` · Estimated effort: `8–10 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:
- Reusable artifact generator producing `eval/` directory per model version
- Artifacts (per `design.md` §4.4):
  - `metrics.json` — primary + secondary, machine-readable
  - `reliability_diagrams.png` — per-class calibration plot (post-calibration)
  - `confusion_matrix.png` — at argmax
  - `segment_metrics.csv` — handedness, park, count, inning, month-of-season
  - `temporal_cv_results.csv` — per-fold metrics
  - `feature_importance.csv` — for tree models (LightGBM); skip for LR
  - `commit_sha.txt` — git commit
  - `data_hash.txt` — training-data hash
- Reused for LR baseline, LightGBM pre-pitch, post-pitch, MLP, LightGBM batted-ball

**OUT of scope**:
- Surfacing artifacts on Ops dashboard → Phase 4e.5
- Comparison artifact (MLP vs LGBM head-to-head) → 2c.9 extends this

---

## Objectives

1. `generate_eval_artifact(model_bundle, cv_result, model_dir)` produces a complete `eval/` directory.
2. Artifacts are deterministic given a seed (image rendering can vary by Matplotlib backend; pin backend).
3. `metrics.json` is machine-readable and consumed by 4e.1 (Ops dashboard).

## Step-by-step implementation tasks

1. Author `_plots.py`:
   - `reliability_diagram(y_true, y_pred_proba, n_bins=10) -> matplotlib.figure.Figure` per class.
   - `confusion_matrix_plot(y_true, y_pred_argmax, classes) -> Figure`.
   - Use Matplotlib `Agg` backend for headless rendering.
2. Author `_segments.py`:
   - `segment_brier(df, segment_cols=['stand','park_id','count','inning','month'])` returns DataFrame.
3. Author `artifact.py`:
   ```python
   def generate(model_bundle, cv_result, predictions_test, model_dir: Path):
       eval_dir = model_dir / "eval"; eval_dir.mkdir(exist_ok=True)
       (eval_dir / "metrics.json").write_text(json.dumps(_summarize(cv_result), indent=2))
       _plots.reliability_diagram(...).savefig(eval_dir / "reliability_diagrams.png")
       _plots.confusion_matrix_plot(...).savefig(eval_dir / "confusion_matrix.png")
       _segments.segment_brier(predictions_test).to_csv(eval_dir / "segment_metrics.csv", index=False)
       _per_fold(cv_result).to_csv(eval_dir / "temporal_cv_results.csv", index=False)
       if isinstance(model_bundle.estimator, lgb.Booster):
           _feature_importance(model_bundle).to_csv(eval_dir / "feature_importance.csv", index=False)
       (eval_dir / "commit_sha.txt").write_text(git_commit_sha() + "\n")
       (eval_dir / "data_hash.txt").write_text(model_bundle.training_data_hash + "\n")
   ```
4. Wire into `train_pre.py` and `train_lr_baseline.py` so each training run emits its own `eval/`.
5. Author `test_artifact.py`: synthetic CVResult → all 8 files exist with non-empty content.
6. Run for both production models (`pitch_outcome_pre` and `pitch_outcome_lr_baseline`); confirm all files present.

## Known edge cases

- **Matplotlib backend**: explicitly set `matplotlib.use('Agg')` at module import — running on a server has no DISPLAY.
- **`segment_metrics.csv` size**: cartesian over (handedness, park, count, inning, month) → ~50K rows. Acceptable; aggregate views are computed in 4e.5.
- **`feature_importance` for LR**: not meaningful (use coefficients magnitude). Skip rather than misinform.
- **Reliability diagram bins**: 10 quantile bins is the default; equal-width also acceptable for high-volume classes.
- **JSON serialization of `numpy.float32`**: cast to `float` explicitly to avoid `TypeError`.
