# 2a.5 — LightGBM train + isotonic calibration

> Owning phase: `phase-2-models/2a-pitch-pre` · Estimated effort: `12–16 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:

- LightGBM multinomial classifier on Tier 1+2+3 features (5-class output: ball/called_strike/swinging_strike/foul/in_play)
- Isotonic regression per class fitted on a temporal holdout (validation year per fold)
- `calibrator.json` capturing per-class isotonic breakpoints
- Training run produces `model.lgb` + `calibrator.json` per fold
- Final production model: train on full 2015–2024, validate on 2025 (no test holdout for production), use fold-4 calibrator transferred — OR re-fit isotonic on 2025 holdout
- Determinism: seed 42, `force_row_wise=True`, `deterministic=True`

**OUT of scope**:

- LR baseline → 2a.6
- Eval artifact generation → 2a.7
- ONNX export + serving → 2a.8
- Forward simulator → 2a.9

---

## Objectives

1. Fold-level training produces a model that beats the LR baseline on Brier (validated in 2a.6).
2. Isotonic calibration reduces ECE by at least 30% over uncalibrated predictions.
3. Reproducibility: same seed → identical `model.lgb` byte-for-byte.
4. Training time per fold ≤ 30 minutes on the workstation CPU.

## Step-by-step implementation tasks

1. Author `_isotonic.py`:
   - Wrapper around `sklearn.isotonic.IsotonicRegression` per class (one-vs-rest).
   - `fit(y_true, y_pred_proba) -> dict[class, IsotonicRegression]`
   - `transform(y_pred_proba) -> y_pred_proba_calibrated` then renormalized to sum to 1.
   - `to_json(path)` / `from_json(path)` serialization.
2. Author `train_pre.py`:
   ```python
   def model_factory(train_df, val_df, seed=42):
       params = dict(objective="multiclass", num_class=5, metric="multi_logloss",
                     learning_rate=0.05, num_leaves=63, min_data_in_leaf=200,
                     feature_fraction=0.8, bagging_fraction=0.8, bagging_freq=5,
                     seed=seed, deterministic=True, force_row_wise=True)
       dtrain = lgb.Dataset(train_df[FEATURES], label=train_df["label"])
       dval = lgb.Dataset(val_df[FEATURES], label=val_df["label"])
       booster = lgb.train(params, dtrain, num_boost_round=2000,
                           valid_sets=[dtrain, dval],
                           callbacks=[lgb.early_stopping(50), lgb.log_evaluation(100)])
       val_pred = booster.predict(val_df[FEATURES])
       calibrator = IsotonicCalibrator.fit(val_df["label"], val_pred)
       return ModelBundle(booster, calibrator)
   ```
3. Run via `cv_harness.run(model_factory, feature_loader, [brier, log_loss, ece])`.
4. Persist outputs:
   ```
   /var/lib/thebullpen/models/pitch_outcome_pre/v<N>/
     ├── model.lgb
     ├── calibrator.json
     ├── metadata.json
     ├── feature_pipeline.json   (covers Tier 1+2+3)
     └── training_data.parquet   (snapshot of features used)
   ```
5. Author `test_train_pre.py`:
   - On 5K-row synthetic dataset, training completes; ECE post-calibration < ECE pre-calibration.
   - Reproducibility: two runs with the same seed produce byte-identical `model.lgb`.
6. Production model (after fold experiments are validated):
   ```bash
   uv run python -m thebullpen.pitch.train_pre --train 2015:2024 --val 2025 --version v1
   ```
7. Inspect `metadata.json`:
   ```json
   {
     "model_name": "pitch_outcome_pre",
   ```

## Known edge cases

- **Class imbalance**: `in_play` is ~14% of pitches, `swinging_strike` ~10%; LightGBM multinomial handles this fine without weighting.
- **Calibrator monotonicity**: isotonic is monotone by construction; renormalization can break monotonicity per-class. Acceptable trade-off — log loss benefits more from valid-distribution outputs than strict monotonicity.
- **Determinism on different machines**: LightGBM is deterministic per machine architecture; cross-machine bit-identical is NOT guaranteed. Document — CI builds on Ubuntu may differ from local Mac.
- **`feature_pipeline.json` hash**: must hash the _schema_ (column names, dtypes, encoding maps), not the data. The hash is what 3a.3 enforces at registration.
