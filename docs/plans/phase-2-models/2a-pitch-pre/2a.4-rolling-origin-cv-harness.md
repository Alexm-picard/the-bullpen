# 2a.4 — Rolling-origin CV harness

> Owning phase: `phase-2-models/2a-pitch-pre` · Estimated effort: `5–7 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:
- A reusable cross-validation harness for any model that consumes the `features` table
- 4 folds spanning 2015–2025 (decision `[56]`):
  | Fold | Train | Validation | Test |
  |---|---|---|---|
  | 1 | 2015–2020 | 2021 | 2022 |
  | 2 | 2015–2021 | 2022 | 2023 |
  | 3 | 2015–2022 | 2023 | 2024 |
  | 4 | 2015–2023 | 2024 | 2025 |
- Within-fold split granularity: by **date**, not by game or pitch (decision `[59]`)
- API: `harness.run(model_factory, feature_loader, eval_metrics) -> CVResult`
- Reports mean ± std-dev across folds (decision `[57]`)

**OUT of scope**:
- Specific model training (LightGBM, LR, MLP) — that's 2a.5/2a.6/2c
- Eval artifact directory generation — `2a.7-eval-artifact-generator.md`

---

## Objectives

1. `harness.run(...)` returns per-fold metrics + summary (mean ± std).
2. Within-fold splits are deterministic given a seed.
3. End-to-end run on a tiny synthetic feature set + a tiny LightGBM finishes in <60s.
4. Wrong-direction guards: an attempt to use `train_test_split` on a fold raises `LeakageError`.

## Step-by-step implementation tasks

1. Author `metrics.py`:
   ```python
   def multiclass_brier(y_true_onehot, y_pred_proba) -> float: ...
   def multiclass_log_loss(y_true_int, y_pred_proba) -> float: ...
   def expected_calibration_error(y_true_onehot, y_pred_proba, n_bins=10) -> float: ...
   ```
2. Author `leakage_guards.py`:
   ```python
   class LeakageError(RuntimeError): pass

   def assert_no_random_split(*args, **kwargs):
       raise LeakageError("Random splits are forbidden — use rolling-origin CV.")
   ```
3. Author `cv_harness.py`:
   ```python
   FOLDS = [
     ((2015,2020), 2021, 2022),
     ((2015,2021), 2022, 2023),
     ((2015,2022), 2023, 2024),
     ((2015,2023), 2024, 2025),
   ]

   class CVResult(NamedTuple):
       per_fold: list[dict[str, float]]
       summary: dict[str, tuple[float, float]]   # (mean, std)

   def run(model_factory, feature_loader, eval_metrics) -> CVResult:
       per_fold = []
       for k, (train_yrs, val_yr, test_yr) in enumerate(FOLDS, 1):
           train = feature_loader(train_yrs[0], train_yrs[1], fold=k)
           val   = feature_loader(val_yr, val_yr, fold=k)
           test  = feature_loader(test_yr, test_yr, fold=k)
           # Within-fold: by date — split val/test by date if needed (here val and test are full-year holdouts)
           model = model_factory(train, val)
           preds = model.predict_proba(test)
           per_fold.append({m.__name__: m(test.y, preds) for m in eval_metrics})
       summary = {k: (mean(v), stdev(v)) for k, v in transpose(per_fold).items()}
       return CVResult(per_fold, summary)
   ```
4. Author `test_cv_harness.py`:
   - Synthetic 200-pitch dataset spanning 2015–2025; assert harness runs 4 folds, returns 4 per-fold dicts.
   - `assert_no_random_split` test: importing it and calling raises.
5. Document the API in module docstring with a usage example for LightGBM (preview of 2a.5).

## Known edge cases

- **Fold 1 train years**: 2015–2020 spans rule changes (no DH in NL pre-2022, etc.). Mention in module docstring; not a leakage issue, just a non-stationarity reminder.
- **Within-fold val + test by date**: in our schema, val and test are full years already, so within-fold splitting is trivial. If we ever need finer splits (e.g., for hyperparameter selection within a fold), enforce by-date.
- **Mean ± std with only 4 samples**: noisy. Honest reporting (decision `[57]`) — disclose `n_folds=4` in eval artifacts.
- **`feature_loader` contract**: must return a dataframe with `.y` and feature columns; document.
- **Seed management**: the harness sets seeds; model factories should accept a seed parameter and use it.
