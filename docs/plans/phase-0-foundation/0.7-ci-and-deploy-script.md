## Objectives

1. Push to `main` triggers Java + frontend CI; both must be green to merge.
2. `./deploy.sh` from a fresh shell pulls latest, builds, restarts, smoke-checks. Failure leaves prior good state running.
3. Frontend Vercel deploy succeeds on `main` push.
4. CI cache hit rate > 50% on subsequent runs (pnpm + Maven).
5. Post-deploy smoke script catches a broken `/v1/health`.

## Step-by-step implementation tasks

1. Author `deploy.sh` per shape in `../00-DEPLOYMENT-STRATEGY.md`. Make executable.
2. Author `scripts/check-no-live-games.sh`:
   ```bash
   #!/usr/bin/env bash
   # Phase 0 placeholder — always allows deploy.
   # Phase 4d will replace this with real MLB schedule checking.
   exit 0
   ```
3. Author `scripts/post-deploy-smoke.sh`:
   ```bash
   #!/usr/bin/env bash
   set -euo pipefail
   for i in {1..10}; do
     if curl -fsS http://localhost:8080/v1/actuator/health \
        && curl -fsS http://localhost:8081/v1/actuator/health; then
       echo "Smoke OK"
       exit 0
     fi
     sleep 3
   done
   echo "Smoke FAILED"
   exit 1
   ```
4. Author `.github/workflows/test-java.yml`:
   ```yaml
   name: test-java
   on: [push, pull_request]
   jobs:
     test:
       runs-on: ubuntu-latest
       services:
         clickhouse:
           image: clickhouse/clickhouse-server:24.3
           ports: ["8123:8123","9000:9000"]
       steps:
         - uses: actions/checkout@v4
         - uses: actions/setup-java@v4
           with:
             distribution: temurin
             java-version: '21'
             cache: maven
         - name: Build + test
           working-directory: backend
           run: ./mvnw -B verify
   ```
5. Author `.github/workflows/test-frontend.yml`:
   ```yaml
   name: test-frontend
   on: [push, pull_request]
   jobs:
     test:
       runs-on: ubuntu-latest
       defaults: { run: { working-directory: frontend } }
       steps:
         - uses: actions/checkout@v4
         - uses: pnpm/action-setup@v3
           with: { version: 9 }
         - uses: actions/setup-node@v4
           with: { node-version: 20, cache: pnpm, cache-dependency-path: frontend/pnpm-lock.yaml }
         - run: pnpm install --frozen-lockfile
         - run: pnpm typecheck
         - run: pnpm test
   ```
6. Push a trivial change. Verify both workflows go green.
7. Verify `./deploy.sh` (locally on the host) end-to-end: edit a comment, push, run `./deploy.sh`, hit `/v1/health` after.
8. Negative test: deliberately break the JAR (e.g. `mv app.jar app.jar.bad`), run `deploy.sh` — confirm it aborts at smoke and `systemctl status` shows the prior version still running.

## Known edge cases

- **WSL `git pull` ownership**: Git's "dubious ownership" warning. Add `git config --global --add safe.directory /home/$USER/thebullpen` once per host.
- **`./mvnw` perms after fresh clone**: ensure `chmod +x mvnw` and tracked in git.
- **Worker restart-before-API**: deliberate — worker should be ready when API starts so `@Scheduled` jobs catch up. Only invert if the API fails to start with worker already serving (it shouldn't).
- **GitHub Actions matrix flakiness with Testcontainers**: provide `services:` clause inline rather than using Testcontainers Docker-in-Docker; faster and more reliable for ClickHouse smoke.
