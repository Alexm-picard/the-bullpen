# 0.8 — Prometheus + Grafana

> Owning phase: `phase-0-foundation` · Estimated effort: `4–6 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:
- Prometheus + node_exporter + Grafana running under systemd-managed Docker
- Prometheus scrapes both Spring units (`/v1/actuator/prometheus`) and node_exporter
- Grafana provisioned with one trivial dashboard ("Application overview"): request rate, p95 latency, JVM heap
- Grafana behind HTTP basic locally on `127.0.0.1:3000`; not exposed via Cloudflare Tunnel
- Provisioning files committed under `infra/prometheus/` and `infra/grafana/`

**OUT of scope**:
- ML Ops dashboard (drift, A/B status, retraining queue) → Phase 3 sub-plans
- System dashboard with deeper detail → can extend the existing one as metrics are added
- Public exposure of Grafana — no.

---

## Objectives

1. Prometheus running, scraping every 15s, retention 30 days.
2. Grafana running, dashboards survive restart (via filesystem provisioning).
3. Application dashboard shows request rate, p95 latency, JVM heap with live data.
4. Restart of the host preserves all metrics + dashboards.
5. Adding a new scrape target requires only a config file change + `systemctl reload prometheus`.

## Step-by-step implementation tasks

1. Generate `GRAFANA_ADMIN_PASSWORD`, append to `secrets.env`.
2. Author `infra/docker/docker-compose.observability.yml`:
   ```yaml
   services:
     prometheus:
       image: prom/prometheus:v2.55.0
       restart: unless-stopped
       volumes:
         - ../prometheus/prometheus.yml:/etc/prometheus/prometheus.yml:ro
         - /var/lib/thebullpen/prometheus:/prometheus
       command: ["--config.file=/etc/prometheus/prometheus.yml",
                 "--storage.tsdb.retention.time=30d",
                 "--web.enable-lifecycle"]
       ports: ["127.0.0.1:9090:9090"]
       mem_limit: 1g

     node-exporter:
       image: prom/node-exporter:v1.8.2
       restart: unless-stopped
       network_mode: "host"
       pid: "host"
       volumes:
         - "/:/host:ro,rslave"
       command: ["--path.rootfs=/host"]

     grafana:
       image: grafana/grafana-oss:11.2.0
       restart: unless-stopped
       environment:
         GF_SECURITY_ADMIN_PASSWORD: ${GRAFANA_ADMIN_PASSWORD}
         GF_USERS_ALLOW_SIGN_UP: "false"
       volumes:
         - /var/lib/thebullpen/grafana:/var/lib/grafana
         - ../grafana/provisioning:/etc/grafana/provisioning:ro
         - ../grafana/dashboards:/var/lib/grafana/dashboards:ro
       ports: ["127.0.0.1:3000:3000"]
       mem_limit: 512m
   ```
3. Author `infra/prometheus/prometheus.yml`:
   ```yaml
   global:
     scrape_interval: 15s
     evaluation_interval: 15s
   scrape_configs:
     - job_name: thebullpen-api
       metrics_path: /v1/actuator/prometheus
       static_configs:
         - targets: ['host.docker.internal:8080']
     - job_name: thebullpen-worker
       metrics_path: /v1/actuator/prometheus
       static_configs:
         - targets: ['host.docker.internal:8081']
     - job_name: node
       static_configs:
         - targets: ['127.0.0.1:9100']
   ```
   Note: in WSL2, `host.docker.internal` works. Add `--add-host=host.docker.internal:host-gateway` to the compose file if needed.
4. Author Grafana datasource provisioning at `infra/grafana/provisioning/datasources/prometheus.yml`:
   ```yaml
   apiVersion: 1
   datasources:
     - name: Prometheus
       type: prometheus
       access: proxy
       url: http://prometheus:9090
       isDefault: true
   ```
5. Author dashboard provisioning at `infra/grafana/provisioning/dashboards/dashboards.yml`:
   ```yaml
   apiVersion: 1
   providers:
     - name: bullpen
       orgId: 1
       folder: 'The Bullpen'
       type: file
       options:
         path: /var/lib/grafana/dashboards
   ```
6. Author `infra/grafana/dashboards/application.json` with three panels:
   - Request rate: `sum(rate(http_server_requests_seconds_count{job="thebullpen-api"}[5m]))`
   - p95 latency: `histogram_quantile(0.95, sum(rate(http_server_requests_seconds_bucket{job="thebullpen-api"}[5m])) by (le))`
   - JVM heap used: `jvm_memory_used_bytes{job="thebullpen-api",area="heap"}`
   Use Grafana's UI to author once, then export JSON, commit it.
7. Author `infra/grafana/dashboards/system.json` showing CPU / memory / disk / network from node_exporter (use a community dashboard JSON as starting point; trim to essentials).
8. Author the systemd unit and enable: `sudo systemctl daemon-reload && sudo systemctl enable --now observability`.
9. Verify:
   - `curl http://127.0.0.1:9090/api/v1/targets` shows all three targets `health=up`.
   - Open `http://127.0.0.1:3000` from Windows browser (port-forwarded via WSL2). Log in with admin + password from `secrets.env`. Application dashboard shows live data.
10. Reboot test: `wsl --shutdown` and re-enter; verify Prometheus retains TSDB, Grafana retains dashboards.

## Known edge cases

- **`host.docker.internal` not resolving inside WSL2 Docker**: add `extra_hosts: ["host.docker.internal:host-gateway"]` to compose. Or use the WSL2 host's IP.
- **node_exporter under WSL**: WSL's `/proc` exposure differs from native Linux. Some metrics (e.g. uptime) may be off; mostly cosmetic.
- **Storage growth**: 30d retention with 15s scrape on a small target set ≈ 1-2 GB. Safe on the SSD.
- **Grafana provisioning**: dashboards committed to git as JSON; do NOT edit through the UI without exporting back. Drift is a defect.
