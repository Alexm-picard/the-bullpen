## Step-by-step implementation tasks

1. From elevated PowerShell: `wsl --install -d Ubuntu-24.04`. Reboot if Windows requests.
2. Set WSL default version: `wsl --set-default-version 2`.
3. Create `C:\Users\<user>\.wslconfig` with:
   ```ini
   [wsl2]
   memory=32GB
   processors=8
   swap=8GB
   localhostForwarding=true
   ```
4. From elevated PowerShell: `wsl --shutdown` then re-enter Ubuntu so the cap applies.
5. Inside WSL, edit `/etc/wsl.conf`:
   ```ini
   [boot]
   systemd=true

   [user]
   default=<your-username>

   [network]
   generateResolvConf=true
   ```
6. From PowerShell: `wsl --shutdown`. Re-enter Ubuntu. Confirm `systemctl is-system-running` reports `running` or `degraded` (not `offline`).
7. Apt update + base tools:
   ```bash
   sudo apt-get update && sudo apt-get -y upgrade
   sudo apt-get install -y curl wget git build-essential ca-certificates jq
   ```
8. Create service user and directories:
   ```bash
   sudo useradd -r -s /usr/sbin/nologin -m -d /home/thebullpen thebullpen
   sudo mkdir -p /opt/thebullpen /var/lib/thebullpen /etc/thebullpen
   sudo chown -R thebullpen:thebullpen /opt/thebullpen /var/lib/thebullpen
   sudo chmod 750 /etc/thebullpen
   ```
9. Verify CUDA passthrough — install NVIDIA's WSL CUDA toolkit per [official docs](https://docs.nvidia.com/cuda/wsl-user-guide/index.html). Then:
   ```bash
   nvidia-smi
   # Must show RTX 4070 Super, driver version, CUDA version
   ```
10. Verify reboot recovery:
    ```bash
    # In PowerShell:
    wsl --shutdown
    # Re-enter Ubuntu:
    systemctl status   # tree healthy
    nvidia-smi         # GPU still visible
    free -h            # Mem total ≤ 32 GB
    ```
11. Author `ops/runbooks/wsl-recovery.md` documenting the 4-step recovery: `wsl --shutdown` → re-enter → `systemctl status` → `nvidia-smi`.

## Known edge cases

- **WSL fails to start systemd**: usually a Windows version too old. Update Windows 11 to latest.
- **CUDA shows no GPU**: typically the host driver is too old for WSL. Update NVIDIA Game Ready Driver on Windows side first; the WSL CUDA toolkit relies on the host driver, not a WSL-side driver.
- **Memory cap ignored**: `.wslconfig` only takes effect after `wsl --shutdown`. A reboot of just the distro doesn't reload it.
- **Permission errors with the `thebullpen` user**: `chown` is fragile across WSL/Windows boundary if the dirs were created by another user. Use `chown -R` after every `mkdir`.
