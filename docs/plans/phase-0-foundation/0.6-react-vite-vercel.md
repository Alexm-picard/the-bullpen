## Step-by-step implementation tasks

1. Scaffold:
   ```bash
   cd thebullpen
   pnpm create vite@latest frontend -- --template react-ts
   cd frontend
   pnpm install
   pnpm add @mantine/core @mantine/hooks @tanstack/react-query
   pnpm add -D tailwindcss postcss autoprefixer openapi-typescript
   pnpm dlx tailwindcss init -p
   ```
2. Configure `tsconfig.json` with `strict: true`, `noUncheckedIndexedAccess: true`, target ES2022.
3. Author `vite.config.ts` exposing the dev server on `5173`. Note: `vite preview` not used.
4. Author `frontend/scripts/regen-openapi.sh`:
   ```bash
   #!/usr/bin/env bash
   set -euo pipefail
   curl -fsSL "${BACKEND_OPENAPI_URL:-http://localhost:8080/v1/v3/api-docs}" \
     | npx openapi-typescript - -o src/api/types.ts
   ```
5. Author `src/api/client.ts`:
   ```ts
   const base = import.meta.env.VITE_API_BASE;
   export async function get<T>(path: string): Promise<T> {
     const res = await fetch(`${base}${path}`, { credentials: 'omit' });
     if (!res.ok) throw new Error(`${res.status} ${res.statusText}`);
     return res.json() as Promise<T>;
   }
   ```
6. Author `src/api/use-health.ts`:
   ```ts
   import { useQuery } from '@tanstack/react-query';
   import { get } from './client';
   export type Health = { status: string; thread: string };
   export const useHealth = () =>
     useQuery({ queryKey: ['health'], queryFn: () => get<Health>('/health') });
   ```
7. Author `src/pages/health-page.tsx` with loading / error / data branches.
8. Wire Mantine + TanStack QueryProvider in `src/main.tsx`.
9. Author backend `CorsConfig`:
   ```java
   @Configuration
   public class CorsConfig {
       @Bean
       WebMvcConfigurer corsConfigurer() {
           return new WebMvcConfigurer() {
               @Override public void addCorsMappings(CorsRegistry r) {
                   r.addMapping("/**")
                    .allowedOriginPatterns(
                        "https://thebullpen.net",
                        "https://*.vercel.app",
                        "http://localhost:5173"
                    )
                    .allowedMethods("GET","POST","OPTIONS")
                    .allowCredentials(false)
                    .maxAge(3600);
               }
           };
       }
   }
   ```
10. Author `vercel.json` at repo root:
    ```json
    {
      "buildCommand": "cd frontend && pnpm install && pnpm build",
      "outputDirectory": "frontend/dist",
      "framework": null,
      "env": { "VITE_API_BASE": "https://api.thebullpen.net/v1" }
    }
    ```
11. Cloudflare DNS: `thebullpen.net` (apex) в†’ `cname.vercel-dns.com.` per Vercel's domain UI. Verify Vercel reports domain attached.
12. Push to `main`. Vercel auto-deploys.
13. Visit `https://thebullpen.net`: expect to see the health page rendering live JSON from the backend.

## Known edge cases

- **Vercel preview deployments**: each PR gets a `*.vercel.app` URL вЂ” the CORS pattern `https://*.vercel.app` covers this. Don't tighten without thinking.
- **`credentials: 'omit'`**: we never send cookies. Auth is HTTP basic on `/admin/*` only and is set per-request manually.
- **OpenAPI codegen drift**: `src/api/types.ts` is gitignored deliberately вЂ” regenerate locally and in CI; never commit it.
- **DNS apex CNAME**: Cloudflare uses CNAME flattening for apex records. If Vercel ever rejects, fall back to `A` records to Vercel's published IPs.
- **WSL2 + localhost**: dev server runs inside WSL on `5173`. Windows host reaches it via `localhost:5173` thanks to `localhostForwarding=true` (set in 0.1).
