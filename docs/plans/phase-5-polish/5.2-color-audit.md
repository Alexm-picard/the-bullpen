## Scope boundaries

**IN scope**:
- `pnpm lint:hex-codes` MUST return zero hits in `src/` outside `src/design/`
- Replace any inline hex codes with token references
- Verify accent (`#B53D2C`) used sparingly (not as a default text color)
- Verify Viridis used wherever data viz appears
- Confirm no `box-shadow` in any component (decision `[107]`); use 1 px borders
- Confirm no `linear-gradient` (decision `[107]`)

**OUT of scope**:
- Adding new color tokens — only do if a real design need emerges (rare)
- A/B testing color palette — locked

**Hard rule** (CLAUDE.md rule 1, decision `[105]`–`[107]`): hex codes in components are defects.

## Step-by-step implementation tasks

1. Run `pnpm lint:hex-codes` and treat each hit as a defect:
   - Replace `#fff` → `colors.bgElevated`.
   - Replace bespoke greens → `colors.viz.viridis[N]`.
   - Replace red → `colors.accent` (only if semantic — alarms, disagreements).
2. Run `grep -rn "box-shadow\|drop-shadow" frontend/src/`. Replace each with a 1px border (`border border-bg-emphasis`).
3. Run `grep -rn "linear-gradient" frontend/src/`. Replace each with a flat color (or remove).
4. Verify accent usage is restrained: `grep -rn "accent" frontend/src/components | wc -l` should be a small number (handful of intentional uses, e.g., disagreement marker, primary CTA).
5. Visual review of data-viz components: confirm Viridis used (not rainbow / not custom).
6. Take screenshots of any visually affected components; document in PR.

## Known edge cases

- **Mantine internal styles** may use hex / shadows — not in our `src/`, won't be flagged. OK.
- **External SVGs** (stadiums) — `stroke="currentColor"` so they inherit theme. No hex inside.
- **3rd-party charting library defaults** — verify Recharts colors are tokens, not defaults.
- **Migration cost**: if many violations exist, allocate the full 4 hours; don't skim.
