**IN scope**:

## Step-by-step implementation tasks

1. Author the template:
   ```markdown
   # Postmortem: <one-line title>

   - **Date range**: YYYY-MM-DD to YYYY-MM-DD
   - **Severity**: page / notice
   - **Affected model**: model_name v<N>
   - **Detected by**: detector job name + threshold
   - **Operator**: <name>
   - **Summary** (3 sentences max)

   ## Timeline (UTC)
   | When | What |
   |---|---|
   | YYYY-MM-DDTHH:MM | Drift detector fired |
   | YYYY-MM-DDTHH:MM | Investigation began |
   | ... | ... |

   ## What happened
   <prose>

   ## Dashboards
   ![Calibration drift](path/to/grafana-snapshot-1.png)
   ![PSI heatmap](path/to/grafana-snapshot-2.png)

   ## Drift metrics that fired
   | Metric | Value | Threshold |
   |---|---|---|
   | calibration_error | 0.052 | 0.030 (1.5× of training 0.020) |

   ## Hypotheses + tests
   1. Hypothesis A: <text>
      - Test: <how we tested>
      - Result: confirmed / rejected
   2. Hypothesis B: ...

   ## Root cause (5-Whys)
   1. Why did calibration drift? Because ...
   2. Why? Because ...
   3. ...
   5. Root cause: <answer>

   ## Resolution
   <what we did>

   ## What changed in the system
   - [ ] Code change: <link to PR>
   - [ ] Test added: <link>
   - [ ] Alert tightened: <details>
   - [ ] Decision logged: `[N]` in decisions.md

   ## Lessons
   <prose>
   ```
2. Document the process:
   - When an alert fires, the operator copies the template, fills it in alongside the investigation, and commits.
   - Postmortems live under `docs/postmortems/` with date-prefixed filenames.
3. Link from README ("Operating evidence" section) and from Ops dashboard ("Postmortems" tab — extension to 4e.5 if time allows; or just in README).
4. **First real postmortem**: when the first drift event lands during operation, follow the template. Honest reporting — don't sanitize.

## Known edge cases

- **Synthetic drift events in production**: when running drills, label the postmortem as "Drill — not real production drift" and store under `docs/postmortems/drills/`.
- **No drift events ever**: unlikely (an MLB season has obvious mid-season shifts: ABS, weather, lineup changes). If by mid-season no drift fired, lower thresholds and re-run synthetic tests to verify the detector isn't broken.
- **Postmortem timing**: don't write too soon. Wait until the resolution is in place. But don't wait too long — memory fades within days.
