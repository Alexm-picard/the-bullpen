## Step-by-step implementation tasks

1. `App.tsx`:
   ```tsx
   const ParksPage = lazy(() => import('./pages/parks/parks-page'));
   const GamePage = lazy(() => import('./pages/game/game-page'));
   const OpsPage = lazy(() => import('./pages/ops/ops-page'));
   const AboutPage = lazy(() => import('./pages/about/about-page'));
   const PlayersPage = lazy(() => import('./pages/players/players-page'));
   <Suspense fallback={<RoutePending/>}>
     <Routes>...</Routes>
   </Suspense>
   ```
2. `vite.config.ts`:
   ```ts
   build: {
     rollupOptions: {
       output: {
         manualChunks: {
           'mantine': ['@mantine/core', '@mantine/hooks'],
           'tanstack': ['@tanstack/react-query'],
           'recharts': ['recharts'],
         }
       }
     }
   }
   ```
3. Author `bundle-budget.mjs`:
   ```js
   const dist = 'dist/assets';
   const initialChunk = await fileSize(`${dist}/index-*.js`);
   if (initialChunk > 300_000) { console.error(`initial chunk ${initialChunk} > 300KB`); process.exit(1); }
   ```
   Run in CI after `pnpm build`.
4. Lighthouse CI: configure `lighthouserc.json` with thresholds Performance 80, Accessibility 90, Best Practices 80, SEO 80. Run against a temp Vercel preview deploy or local production build.
5. Audit pass: run `pnpm build && pnpm preview`; profile with Chrome DevTools; identify large modules; tree-shake or replace.
6. Image optimization: ensure all SVGs use `currentColor` and minimal markup; convert any PNGs to WebP if any exist.
7. Document the perf baseline (chunk sizes per page) in PR.

## Known edge cases

- **Mantine bundle weight**: ~80 KB gz for what we use. Tree-shake by importing specific components: `import { Slider } from '@mantine/core'` works.
- **Recharts is heavy** (~70 KB). Consider lighter alternative or load-on-demand for Ops drift charts.
- **Lighthouse on local dev server**: slow; always test against a production build (`pnpm build && pnpm preview`).
- **CI Lighthouse flakiness**: allow ±5 points tolerance per metric.
