   ...

2. Write each section. Be concise. Link aggressively rather than duplicate text.
3. Take a clean Park Explorer screenshot for the hero.
4. Author or refine the architecture diagram (start from `design.md` §2).
5. Verify every link resolves: `pnpm exec markdown-link-check README.md` (add the dep).

## Known edge cases

- **Domain mismatch**: ensure README links use `https://thebullpen.net`, not `localhost`.
- **Sensitive content in screenshots**: nothing should leak admin credentials, B2 keys, etc. Audit before commit.
- **Launch posts as marketing**: keep technical, not promotional. Recruiters and engineers come from r/programming and HN; don't oversell.
- **Server load**: if HN/Reddit traffic spikes, Cloudflare handles. The Spring API is rate-limited to 60 req/min/IP (Risk Register I2).
