## Step-by-step implementation tasks

1. Add `framer-motion`: `pnpm add framer-motion`.
2. Wrap thumbnail grid with `<motion.div layout />` so reorder transitions animate (200 ms, ease).
3. Annotate page: at the top, a single sentence explaining "Drag the sliders to see how an at-bat at this speed/angle/spray would play in each park." Source Serif italic, restrained.
4. Tooltip refinement: position above thumbnail (not floating randomly); use `colors.bgElevated` background, 1px `bg-emphasis` border, no shadow.
5. Empty state: when all P(HR) < 0.005, render a placeholder explaining "no realistic HR scenarios at these inputs."
6. Re-render performance: profile with React DevTools; ensure re-renders happen only on changed thumbnails (memo by `(parkId, probHr.toFixed(3))`).
7. Take side-by-side screenshots (4c.4 vs 5.5) — attach to PR. Recruiter test: would you screenshot this for a friend? If not, iterate again.
8. Lighthouse: confirm no regression (perf score still ≥ 80).

## Known edge cases

- **Framer Motion bundle weight**: ~50 KB gz. Acceptable for the marquee; tree-shake to use only `motion.div` and `AnimatePresence`.
- **Reorder during slider drag**: lock reorder until slider commit (200 ms debounced). Avoid jitter.
- **Animation accessibility**: respect `prefers-reduced-motion` — disable Framer animations when set.
- **When to stop iterating**: if you've spent 14 h and it still feels "fine" not "memorable", commit what you have and document the gap as a v1.5 polish target.
