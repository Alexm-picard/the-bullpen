# 5.1 — Typography pass

> Owning phase: `phase-5-polish` · Estimated effort: `4–6 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:

- Sweep all 5 pages (Game, Player, Park Explorer, Ops, About) verifying:
  - Type scale obeyed (sizes from `tokens.typography.scale` only)
  - Tabular figures on for all numeric data (`font-feature-settings: 'tnum' 1`)
  - Monospace and serif never co-occur on the same line (decision: editorial discipline)
  - Source Serif headlines on About + Park Explorer marquee at 48–64 px
  - Line heights 1.5 body / 1.2 display
- Adjust outliers in component files; document any token-level changes in `tokens.ts`

**OUT of scope**:

- Color audit → 5.2
- Spacing audit → bundled into this leaf
- Accessibility audit → 5.4

---

## Objectives

1. No hard-coded font sizes in components — every `text-*` class consumes a token.
2. Numeric data uses tabular figures everywhere (statistics, table cells).
3. About + Park Explorer feature serif headlines visibly at intended sizes.
4. Manual screenshot review across all 5 pages: typography reads as one cohesive system.

## Step-by-step implementation tasks

1. Add to `frontend/src/design/global.css`:
   ```css
   body {
     font-feature-settings: "tnum" 1;
   }
   .nominal {
     font-feature-settings: "tnum" 0;
   } /* opt-out for purely decorative numerals */
   ```
2. Walk every page; check every `<text className="...">` for hard-coded sizes; replace with `text-scale-N` utilities.
3. Verify monospace + serif separation:
   - About headline (serif) — body never contains a `<code>` or `<span class="font-data">` in the same line.
   - If a violation exists, restructure so the monospace data is on its own line.
4. Verify Source Serif renders boldly: visit `/about` and Park Explorer marquee header in the browser; visually confirm.
5. Take "before" screenshots from a previous commit; compare to "after" — document the deltas in PR description.
6. Run Lighthouse on all 5 pages — typography-related a11y issues should be zero.

---

## Known edge cases

- **`<code>` inline in About prose**: rare, but if it happens, ensure the surrounding line is sans-serif body, not serif headline.
- **Type-scale rounding**: if a page genuinely needs a size between two scale stops, propose adding a new stop to `tokens.ts` (and update Tailwind config).
- **Numerals in icons / glyphs**: tabular figures don't apply; that's fine.
