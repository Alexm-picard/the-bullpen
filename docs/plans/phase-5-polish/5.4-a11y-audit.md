## Step-by-step implementation tasks

1. Add axe-core: `pnpm add -D @axe-core/playwright axe-html-reporter`.
2. Author `frontend/scripts/run-axe.mjs`:
   ```js
   import { chromium } from "playwright";
   import AxeBuilder from "@axe-core/playwright";
   const PAGES = ["/", "/players", "/parks", "/game/dummy", "/ops", "/about"];
   for (const path of PAGES) {
     const page = await ctx.newPage();
     await page.goto(BASE + path);
     const results = await new AxeBuilder({ page }).analyze();
     const fatals = results.violations.filter(
       (v) => v.impact === "critical" || v.impact === "serious",
     );
     if (fatals.length) {
       console.error(path, fatals);
       process.exitCode = 1;
     }
   }
   ```
3. Run; iterate on findings:
   - Missing labels on form controls — fix
   - Missing alt on images — fix
   - Insufficient contrast — adjust tokens IF necessary, or wrap text in higher-contrast color (rare; audit usually passes given the warm-neutral palette)
   - Focus traps on modals — Mantine's defaults should work; verify
4. Manual keyboard test:
   - Tab through every interactive element; verify focus visible.
   - Park Explorer: implement keyboard navigation among 30 thumbnails (arrow keys to move focus, Enter to open detail).
5. Screen-reader spot-check (NVDA on Windows or VoiceOver on Mac): visit About + Park Explorer; ensure semantic structure makes sense.
6. CI: add axe job to `test-frontend.yml`. Fails on any critical/serious finding.

- **Modal focus traps**: Mantine `Modal` traps focus correctly. Verify.
