## Step-by-step implementation tasks

1. Author `BattedBallRequest`:
   ```java
   public record BattedBallRequest(
       @Min(40) @Max(130) Float launchSpeedMph,
       @Min(-30) @Max(60) Float launchAngleDeg,
       @Min(50) @Max(110) Float releaseSpeedMph,
       @NotBlank String parkId,
       @Pattern(regexp="L|R|S") String stand
   ) {}
   ```
2. Author `PredictionResponse`:
   ```java
   public record PredictionResponse(float probHr, String modelVersion, long latencyMicros, String traceId) {}
   ```
3. Author `PredictBattedBallController`:

   ```java
   @RestController
   @Profile("api")
   @RequestMapping("/predict")
   public class PredictBattedBallController {
     private final ToyBattedBallInference inference;
     private final InferenceMetrics metrics;
     public PredictBattedBallController(ToyBattedBallInference i, InferenceMetrics m) { this.inference=i; this.metrics=m; }

     @PostMapping("/batted-ball")
     public PredictionResponse predict(@Valid @RequestBody BattedBallRequest req) {
       var sample = metrics.startTimer("_toy_batted_ball");
       try {
         float prob = inference.predict(req);
         metrics.incrementCounter("_toy_batted_ball", "champion");
         return new PredictionResponse(prob, "_toy/v0", sample.stop(metrics.timer("_toy_batted_ball")), MDC.get("traceId"));
       } catch (Exception e) {
         metrics.incrementError("_toy_batted_ball", e.getClass().getSimpleName());
         throw e;
       }
     }
   }
   ```

4. Author `ApiErrorAdvice` mapping `MethodArgumentNotValidException` → 400 with the shape, `Exception` → 500 with the shape (no stack-trace leakage, just `code=internal_error`).
5. Author `InferenceMetrics`:
   ```java
   @Component
   public class InferenceMetrics {
     private final MeterRegistry registry;
     private final Map<String, Timer> timers = new ConcurrentHashMap<>();
     // helper methods returning Timer.Sample, Counter, etc.
   }
   ```
6. Author `WarmupReadiness`:
   ```java
   @Component
   @Profile("api")
   public class WarmupReadiness implements ApplicationListener<ApplicationReadyEvent> {
     private final ToyBattedBallInference inference;
     private final ApplicationAvailability availability;
     // on ready: fire 3 dummy predictions, then publish ReadinessState.ACCEPTING_TRAFFIC
   }
   ```
   Use `AvailabilityChangeEvent.publish(...)`.
7. Author `PredictBattedBallControllerTest` with `MockMvc`:
   - 200 path: valid payload returns `probHr` in `[0,1]`.
   - 400 path: missing field → 400 with `error.code=validation_failed`.
   - 415 path: text content type → 415.
8. Smoke test against deployed instance:
   ```bash
   curl -X POST -H 'Content-Type: application/json' \
     -d '{"launchSpeedMph":105,"launchAngleDeg":28,"releaseSpeedMph":94,"parkId":"NYA","stand":"R"}' \
     https://api.thebullpen.net/v1/predict/batted-ball
   ```
   Expect 200 with non-trivial `probHr`.
9. Verify Grafana panel for `prediction_latency_seconds` populates after 5 requests.

## Known edge cases

- **Stand `'S'` (switch hitter)**: feature pipeline can't represent it. Validate on the controller side: switch hitters are `'L'` or `'R'` per matchup; reject `'S'` until Phase 2 introduces the matchup-aware pipeline. (Or default to `'R'` with a warning — choose the strict path.)
- **Unknown `parkId`**: reject with 400 (`unknown_park`). The `park_hr_rate.json` lookup table is a closed set.
- **Cold start**: without `WarmupReadiness`, the first request takes ~2 seconds (ONNX session build + JIT). Uptime Robot would page on this without warm-up (its 2-failure debounce would still catch a sustained cold-start spike across two consecutive 5 min checks).
- **`MDC.get("traceId")`**: requires OpenTelemetry filter wired upstream of controller — verify in 0.3 outcome.
- **Float vs Double in JSON**: Spring Jackson serializes `float` as `0.123456...` precision. Acceptable for v1.
