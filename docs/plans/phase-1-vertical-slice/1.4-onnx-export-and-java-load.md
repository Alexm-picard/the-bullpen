## Step-by-step implementation tasks

1. Add to `backend/pom.xml`:
   ```xml
   <dependency>
     <groupId>com.microsoft.onnxruntime</groupId>
     <artifactId>onnxruntime</artifactId>
     <version>1.18.x</version>
   </dependency>
   ```
2. Add Python deps: `uv add onnx onnxruntime onnxmltools skl2onnx hummingbird-ml`. (Choose ONE conversion path; recommend `onnxmltools.convert.convert_lightgbm`.)
3. Author `export_toy_onnx.py`:
   ```python
   import lightgbm as lgb, onnxmltools, json
   from onnxmltools.convert.common.data_types import FloatTensorType
   booster = lgb.Booster(model_file="/var/lib/thebullpen/models/_toy/v0/model.lgb")
   # Numeric features get a single Float input; we pre-scale before invoking.
   initial_types = [("input", FloatTensorType([None, 5]))]
   onnx_model = onnxmltools.convert.convert_lightgbm(booster, initial_types=initial_types, zipmap=False)
   onnxmltools.utils.save_model(onnx_model, "/var/lib/thebullpen/models/_toy/v0/model.onnx")

   pipeline = {
     "model_name": "_toy_batted_ball",
     "version": "v0",
     "feature_order": FEATURES,
     "preprocess": {
       "park_id_encoded": {"type": "target_encoding", "table_path": "park_hr_rate.json"},
       "stand_is_left":   {"type": "boolean", "from": "stand", "value": "L"}
     },
     "feature_schema_hash": sha256_of_pipeline_json()
   }
   Path(".../feature_pipeline.json").write_text(json.dumps(pipeline, indent=2))
   ```
4. Generate fixture: pick 1 row from real 2024 data, capture its `pitches` columns, run it through Python + ONNX inference, write expected output:
   ```python
   sess = ort.InferenceSession("model.onnx")
   x = np.array([[launch_speed, launch_angle, release_speed, park_enc, stand_left]], dtype=np.float32)
   p = sess.run(None, {"input": x})[0]
   ```
   Save `parity_toy_001.json` (raw input columns) and `parity_toy_001_expected.json` (preprocessed feature vector + final prob).
5. Author `OnnxModel`:
   ```java
   public class OnnxModel implements AutoCloseable {
     private final OrtEnvironment env;
     private final OrtSession session;
     public OnnxModel(Path modelPath) throws OrtException {
       this.env = OrtEnvironment.getEnvironment();
       this.session = env.createSession(modelPath.toString(), new OrtSession.SessionOptions());
     }
     public float[] predictBatch(float[][] features) throws OrtException {
       try (OnnxTensor t = OnnxTensor.createTensor(env, features)) {
         try (OrtSession.Result r = session.run(Map.of("input", t))) {
           float[][] out = (float[][]) r.get(0).getValue();
           return Arrays.stream(out).mapToDouble(a -> a[1]).toArray()  // p(HR)
                  .pipe(arr -> { float[] f = new float[arr.length]; ... });
         }
       }
     }
     @Override public void close() { session.close(); env.close(); }
   }
   ```
6. Author `FeaturePipelineJava` reading `feature_pipeline.json` and exposing `transform(PitchContext) -> float[5]`:
   - Looks up `park_hr_rate.json` for target-encoded park value.
   - Computes `stand_is_left = stand.equals("L") ? 1.0f : 0.0f`.
   - Numeric pass-through for the three Statcast features.
7. Author `ToyBattedBallInference` (`@Component`, `@Profile("api")`) wiring `OnnxModel` + `FeaturePipelineJava`. Loads the model at startup; warm-up firing 3 dummy predictions on `ApplicationReadyEvent`.
8. Author `ToyParityTest`:
   ```java
   @Test
   void parity_with_python() throws Exception {
     var fixture = loadFixture("parity_toy_001.json");
     var expected = loadExpected("parity_toy_001_expected.json");
     var pred = inference.predict(fixture);
     assertThat(pred).isCloseTo(expected.probHr(), within(1e-6));
   }
   ```
9. Author `test_python_java_parity.py` mirror: loads same fixture and same expected, runs Python ONNX, asserts match.
10. Run both. Fix any drift. Drift > 1e-6 means the Java preprocessing diverges from Python's — investigate before continuing.

## Known edge cases

- **`zipmap=False` is essential**: otherwise the LightGBM ONNX output is a list of dicts, not a tensor. ORT-Java has no native ZipMap support.
- **Float32 vs Float64**: LightGBM trains on float64 by default but ONNX export prefers float32. Force `dtype=np.float32` on inputs in both Python and Java; otherwise rounding diverges.
- **`launch_speed_mph IS NULL`**: the toy model assumes non-null. Java pipeline must reject nulls with a clear error; downstream leaves can introduce imputation later.
- **`OrtEnvironment` singletons**: don't create one per request. Spring `@Bean` wraps the session; reuse it.
- **ONNX opset version**: pin to 17 or higher (LightGBM converter requires modern opset).
- **`park_hr_rate.json` divergence**: this lookup file MUST be regenerated whenever `_features_toy.py` retrains. Document in the script.
