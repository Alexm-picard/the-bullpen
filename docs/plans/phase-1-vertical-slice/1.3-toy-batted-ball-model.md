# 1.3 — Toy batted-ball model

> Owning phase: `phase-1-vertical-slice` · Estimated effort: `4–6 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:
- A LightGBM binary classifier on a tiny feature set (5 features) predicting **HR vs. not-HR** for batted balls
- Single output, no per-park heads, no calibration, no eval artifact
- Deliberately under-engineered — pure plumbing for ONNX export and Java load
- Train script `train_toy_batted_ball.py` writes `model.lgb` and a tiny `metadata.json`
- Does NOT touch the registry; output goes to `/var/lib/thebullpen/models/_toy/v0/`

**OUT of scope**:
- 5-class outcome (HR/3B/2B/1B/out) — Phase 2c.5
- 30-park multi-output MLP — Phase 2c.5
- Calibration — Phase 2a.5 / 2c.6
- Eval artifact directory — Phase 2a.7

---

## Objectives

1. A minimum viable model exists and can be loaded by ONNX Runtime in Java (in 1.4).
2. Training is reproducible: same seed → same `model.lgb` byte-for-byte.
3. AUC ≥ 0.7 on held-out test split — sanity that the features carry signal.
4. Training run takes < 60 seconds on the workstation CPU.

## Step-by-step implementation tasks

1. Add deps: `uv add lightgbm scikit-learn`.
2. Author `_features_toy.py`:
   ```python
   FEATURES = [
       "launch_speed_mph",
       "launch_angle_deg",
       "release_speed_mph",  # mild signal: HRs slightly correlate with pitch velo
       "park_id_encoded",     # int target-encoded by 2024 HR rate (computed once)
       "stand_is_left",       # 1 if batter L, 0 otherwise
   ]

   def load_training_frame(client, year: int) -> pd.DataFrame:
       q = f"""
           SELECT launch_speed_mph, launch_angle_deg, release_speed_mph,
                  park_id, stand,
                  toUInt8(events = 'home_run') AS is_hr
           FROM pitches FINAL
           WHERE description = 'in_play' AND toYear(game_date) = {year}
             AND launch_speed_mph IS NOT NULL AND launch_angle_deg IS NOT NULL
       """
       df = pd.DataFrame(client.execute(q), columns=["launch_speed_mph","launch_angle_deg","release_speed_mph","park_id","stand","is_hr"])
       df["stand_is_left"] = (df["stand"] == "L").astype(int)
       hr_rate = df.groupby("park_id")["is_hr"].mean().to_dict()
       df["park_id_encoded"] = df["park_id"].map(hr_rate).astype("float32")
       return df[FEATURES + ["is_hr"]]
   ```
3. Author `train_toy.py`:
   ```python
   import lightgbm as lgb
   from sklearn.model_selection import train_test_split

   def main():
       df = load_training_frame(make_client(), year=2024)
       train, test = train_test_split(df, test_size=0.2, random_state=42, stratify=df["is_hr"])
       params = dict(objective="binary", metric="auc", learning_rate=0.05, num_leaves=31, seed=42, deterministic=True, force_row_wise=True)
       dtrain = lgb.Dataset(train[FEATURES], label=train["is_hr"])
       model = lgb.train(params, dtrain, num_boost_round=200)
       auc = roc_auc_score(test["is_hr"], model.predict(test[FEATURES]))
       log.info("toy model trained", auc=auc, n_train=len(train), n_test=len(test))
       assert auc >= 0.7, f"AUC too low: {auc}"
       outdir = Path("/var/lib/thebullpen/models/_toy/v0")
       outdir.mkdir(parents=True, exist_ok=True)
       model.save_model(str(outdir / "model.lgb"))
       (outdir / "metadata.json").write_text(json.dumps({
         "model_name": "_toy_batted_ball",
         "version": "v0",
         "features": FEATURES,
         "auc": float(auc),
         "git_commit": git_commit_sha(),
         "trained_at": datetime.utcnow().isoformat()
       }, indent=2))
   ```
4. Run: `uv run python -m thebullpen.battedball.train_toy`.
5. Author `test_train_toy.py`:
   - With a synthetic 200-row DataFrame that has clearly separable HR signal, `train_toy` produces a model with `auc > 0.95` (smoke).
6. Document the toy model's *intentional* limitations in module docstring: not calibrated, no per-park split, not registered, not for serving production traffic — strictly Phase-1 plumbing.

## Known edge cases

- **Class imbalance**: HR rate is ~5–7% of in-play balls. AUC handles imbalance OK; log loss would not without class weighting. Acceptable for v1 toy.
- **Park-encoding leakage**: target-encoding `park_id` using the same year's HR rate is mild leakage — fine for the toy. Phase 2a.1 introduces the real out-of-fold target-encoding pipeline.
- **`force_row_wise=True`**: needed for determinism; otherwise LightGBM's parallel tree construction is non-deterministic.
- **Determinism gotcha**: pandas `groupby().mean()` is deterministic for fixed input order, but `train_test_split` requires `random_state=42` set explicitly — already done.
- **Don't register this model**: `_toy` prefix is the convention to keep it out of the registry namespace.
