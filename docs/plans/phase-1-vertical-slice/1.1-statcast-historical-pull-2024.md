# 1.1 — Statcast 2024 historical pull

> Owning phase: `phase-1-vertical-slice` · Estimated effort: `5–7 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:

- Python project skeleton under `ml/` with `pyproject.toml`, `uv` dependency mgmt
- pybaseball pulls 2024 regular-season Statcast in monthly chunks
- Each chunk streamed to ClickHouse `raw_statcast` via clickhouse-driver native protocol
- Pull is idempotent on a (start_date, end_date) range
- Per-chunk SQL assertion: row count within ±5% of published month totals
- Logs include chunk progress, durations, and a final summary line

**OUT of scope**:

- 2015–2023 backfill — happens during Phase 2a's full pipeline build
- Cleaning into `pitches` → `1.2-pitches-cleaning-and-dedup.md`
- Live-game polling → Phase 4d (game state machine) and ingest plumbing in Phase 3
- Weather pulls → deferred to Phase 2 weather feature work

---

## Objectives

1. `ml/thebullpen/ingest/statcast_pull.py --season 2024` writes ~700K rows to ClickHouse `raw_statcast`.
2. Re-running the same command leaves row count unchanged (idempotency).
3. Memory footprint stays under ~1 GB during the run (Risk Register G12 path proven).
4. Total wall time documented (typical: 30–90 minutes depending on Statcast availability).

## Step-by-step implementation tasks

1. Bootstrap Python project:
   ```bash
   cd ml
   uv init --package
   uv add pybaseball clickhouse-driver pandas pyarrow structlog click
   uv add --dev pytest pytest-cov ruff mypy
   ```
   Pin Python 3.11+. Configure `ruff` (line-length 100, target-version py311). Configure `mypy` strict for `thebullpen/`.
2. Author `db/migration/clickhouse/V002__raw_statcast.sql`:
   ```sql
   CREATE TABLE IF NOT EXISTS raw_statcast (
     game_pk          UInt64,
     game_date        Date,
     at_bat_index     UInt16,
     pitch_number     UInt8,
     pitcher          UInt32,
     batter           UInt32,
     pitch_type       LowCardinality(String),
     release_speed    Nullable(Float32),
     release_pos_x    Nullable(Float32),
     release_pos_z    Nullable(Float32),
     plate_x          Nullable(Float32),
     plate_z          Nullable(Float32),
     description      LowCardinality(String),
     events           LowCardinality(String),
     launch_speed     Nullable(Float32),
     launch_angle     Nullable(Float32),
     hc_x             Nullable(Float32),
     hc_y             Nullable(Float32),
     home_team        FixedString(3),
     away_team        FixedString(3),
     stand            FixedString(1),
     p_throws         FixedString(1),
     -- ... add the rest of the Statcast columns we will use
     ingested_at      DateTime DEFAULT now()
   ) ENGINE = MergeTree()
   PARTITION BY toYYYYMM(game_date)
   ORDER BY (game_date, game_pk, at_bat_index, pitch_number);
   ```
3. Author `_clickhouse_client.py` returning a `clickhouse_driver.Client` initialized from env vars (`CLICKHOUSE_*`). Provide `insert_dataframe(table, df)` that uses native protocol (faster than HTTP).
4. Author `_assertions.py` with:
   - `assert_row_count_in_range(table, where_clause, expected, tol_pct=5)`
   - `assert_no_null_pks(table, where_clause)`
5. Author `statcast_pull.py`:
   ```python
   @click.command()
   @click.option("--season", type=int, required=True)
   @click.option("--start-month", type=int, default=3)  # March
   @click.option("--end-month", type=int, default=10)   # October (regular season)
   def main(season, start_month, end_month):
       client = make_client()
       for month in range(start_month, end_month + 1):
           start, end = month_window(season, month)
           log.info("pulling chunk", season=season, month=month, start=str(start), end=str(end))
           df = pybaseball.statcast(start_dt=str(start), end_dt=str(end))
           if df.empty:
               log.warning("empty chunk", month=month); continue
           df = normalize_columns(df)
           client.insert_dataframe("raw_statcast", df)
           log.info("inserted chunk", month=month, rows=len(df))
       run_assertions(client, season)
   ```
   Use small `fetchmany`-style pattern; do NOT load full season into memory.
6. Idempotency — choose one strategy and document it:
   - **Easiest**: pre-delete the partition (`ALTER TABLE raw_statcast DROP PARTITION 'YYYYMM'`) before re-loading that month.
   - **Pick this** for v1; document in module docstring.
7. Author `test_statcast_pull.py` with a small fixture (5 rows) and an in-memory ClickHouse via Testcontainers (or skip ClickHouse and unit-test `normalize_columns` directly).
8. Author `.github/workflows/test-python.yml`:
   ```yaml
   name: test-python
   on: [push, pull_request]
   jobs:
     test:
       runs-on: ubuntu-latest
       defaults: { run: { working-directory: ml } }
       steps:
         - uses: actions/checkout@v4
         - uses: astral-sh/setup-uv@v3
         - name: Install
           run: uv sync --frozen
         - run: uv run ruff check .
         - run: uv run mypy thebullpen
         - run: uv run pytest -q
   ```
9. Apply migration: restart `thebullpen-api` to run V002. Verify `SHOW TABLES` lists `raw_statcast`.
10. Run the pull: `uv run python -m thebullpen.ingest.statcast_pull --season 2024`. Watch logs.
11. Verify row count: `SELECT count(*) FROM raw_statcast WHERE game_date >= '2024-03-28'` ≈ 700K (±5%).

## Known edge cases

- **pybaseball flakiness**: sometimes returns 5xx during peak hours; wrap pulls in retry-with-backoff (3 attempts, 30s/60s/120s).
- **Schema drift**: pybaseball columns occasionally renamed. `normalize_columns` should emit a warning if expected columns are missing rather than KeyError.
- **All-Star break**: mid-July gap; `--end-month=10` covers postseason teams that played late. Document we exclude playoffs in v1 and only include regular-season games (filter by `game_type='R'`).
- **Time zones in `game_date`**: keep as `Date` (no time component); MLB's published date is the home team's local date.
- **Doubleheaders**: `game_pk` is unique per game, so no special handling needed.
