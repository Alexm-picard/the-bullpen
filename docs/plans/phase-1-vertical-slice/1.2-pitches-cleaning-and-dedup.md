# 1.2 — `pitches` cleaning and dedup

> Owning phase: `phase-1-vertical-slice` · Estimated effort: `4–6 h` · Authored: 2026-05-10

---

## Scope boundaries

**IN scope**:
- ClickHouse `pitches` table with cleaned, typed schema
- ReplacingMergeTree dedup keyed on `(game_id, at_bat_index, pitch_number)`
- Cleaning SQL `transform_raw_to_pitches.sql` with explicit type coercions, range checks, enum normalizations
- Per-stage assertions: row counts, NULL PK guard, value-range checks (exit_velocity, launch_angle bounded)
- A small Python orchestrator `transform_pitches.py` that runs the SQL and assertions
- Documented mapping from `raw_statcast` columns to `pitches` columns

**OUT of scope**:
- `pitches_live` (live-game schema) — Phase 4d ingest leaf
- `features` table — Phase 2a feature pipeline
- Weather columns — Phase 2 (after weather pulls land)

---

## Objectives

1. `SELECT count(*) FROM pitches FINAL WHERE toYear(game_date)=2024` matches `SELECT count(*) FROM raw_statcast WHERE toYear(game_date)=2024` modulo dedup.
2. Re-running the transform is idempotent (no duplicate rows, no count drift).
3. Assertion failures abort with a non-zero exit code AND a Discord notice (via `DiscordAlerter` if invoked from a wrapped script).
4. A deliberately injected duplicate row gets collapsed by `FINAL` queries.

## Step-by-step implementation tasks

1. Author `V003__pitches.sql`:
   ```sql
   CREATE TABLE IF NOT EXISTS pitches (
     game_id          UInt64,                     -- aliased from game_pk
     game_date        Date,
     at_bat_index     UInt16,
     pitch_number     UInt8,
     pitcher_id       UInt32,
     batter_id        UInt32,
     pitch_type       LowCardinality(String),     -- normalized enum (FF/SL/CU/CH/...)
     description      Enum8('ball'=0,'called_strike'=1,'swinging_strike'=2,'foul'=3,'in_play'=4,'unknown'=99),
     events           LowCardinality(String),
     release_speed_mph Nullable(Float32),
     plate_x_in       Nullable(Float32),
     plate_z_in       Nullable(Float32),
     launch_speed_mph Nullable(Float32),
     launch_angle_deg Nullable(Float32),
     home_team        FixedString(3),
     away_team        FixedString(3),
     park_id          LowCardinality(String),     -- derived from home_team
     stand            FixedString(1),
     p_throws         FixedString(1),
     balls            UInt8,
     strikes          UInt8,
     outs             UInt8,
     inning           UInt8,
     base_state       UInt8,                       -- bitmask: 1=1B,2=2B,4=3B
     ingested_at      DateTime DEFAULT now()
   ) ENGINE = ReplacingMergeTree(ingested_at)
   PARTITION BY toYYYYMM(game_date)
   ORDER BY (game_date, game_id, at_bat_index, pitch_number);
   ```
2. Author `transform_raw_to_pitches.sql`:
   ```sql
   INSERT INTO pitches
   SELECT
     game_pk AS game_id,
     game_date,
     at_bat_index,
     pitch_number,
     pitcher AS pitcher_id,
     batter AS batter_id,
     upper(pitch_type) AS pitch_type,
     multiIf(
       description IN ('ball','blocked_ball','intent_ball','pitchout'), 'ball',
       description IN ('called_strike'), 'called_strike',
       description IN ('swinging_strike','swinging_strike_blocked','missed_bunt'), 'swinging_strike',
       description IN ('foul','foul_tip','foul_bunt'), 'foul',
       description IN ('hit_into_play','hit_into_play_no_out','hit_into_play_score'), 'in_play',
       'unknown'
     ) AS description,
     events,
     release_speed AS release_speed_mph,
     plate_x AS plate_x_in,
     plate_z AS plate_z_in,
     launch_speed AS launch_speed_mph,
     launch_angle AS launch_angle_deg,
     home_team,
     away_team,
     home_team AS park_id,    -- 1:1 in v1; refine in 2c.3 (park geometry)
     stand,
     p_throws,
     balls, strikes, outs, inning,
     -- base_state encoding:
     toUInt8((on_1b > 0) * 1 + (on_2b > 0) * 2 + (on_3b > 0) * 4) AS base_state,
     now() AS ingested_at
   FROM raw_statcast
   WHERE game_pk NOT IN (SELECT DISTINCT game_id FROM pitches WHERE game_date = raw_statcast.game_date);
   -- Idempotency: ReplacingMergeTree handles re-loads; the WHERE clause keeps inserts cheap.
   ```
3. Author `assertions_pitches.sql` runnable per chunk via Python:
   - `SELECT count(*) FROM pitches WHERE toYear(game_date)=$year` returns expected ±5%.
   - `SELECT count(*) FROM pitches WHERE game_id=0 OR pitcher_id=0` = 0 (PK guard).
   - `SELECT count(*) FROM pitches WHERE launch_speed_mph NOT BETWEEN 40 AND 130 AND launch_speed_mph IS NOT NULL` = 0.
   - `SELECT count(*) FROM pitches WHERE pitch_type NOT IN ('FF','SL','CU','CH','SI','FC','KC','FS','CS','FO','SC','EP','PO','UN','')` = 0.
   - `SELECT count(*) FROM pitches WHERE description = 'unknown'` < 100 (fewer than 100 unrecognized descriptions).
4. Author `transform_pitches.py`:
   ```python
   @click.command()
   @click.option("--year", type=int, required=True)
   def main(year):
       client = make_client()
       sql = load_sql("transform_raw_to_pitches.sql")
       client.execute(sql)
       run_assertions(client, year)
       counts = client.execute(f"""
         SELECT
           (SELECT count(*) FROM raw_statcast WHERE toYear(game_date)=%(y)s) AS raw,
           (SELECT count(*) FROM pitches FINAL WHERE toYear(game_date)=%(y)s) AS clean
       """, {"y": year})
       log.info("done", **dict(zip(["raw","clean"], counts[0])))
   ```
5. Apply migration. Run `uv run python -m thebullpen.ingest.transform_pitches --year 2024`.
6. Verify counts roughly match. Insert a duplicate by hand:
   ```sql
   INSERT INTO pitches SELECT * FROM pitches WHERE game_id=<known> LIMIT 1;
   SELECT count(*) FROM pitches WHERE game_id=<known>;
   SELECT count(*) FROM pitches FINAL WHERE game_id=<known>;
   ```
   Confirm the FINAL count is unchanged (dedup works).
7. Add `test_transform_pitches.py`:
   - Inject 5 rows into `raw_statcast`, run transform, assert `pitches FINAL` returns 5.
   - Inject a duplicate, assert `pitches FINAL` still returns 5.
   - Inject an out-of-range `launch_speed`, assert assertion sweep raises.

## Known edge cases

- **`description` mapping ambiguity**: Statcast uses `pitchout` and `intent_ball` as variants of ball; we collapse them. If we ever care about differentiating, it'll be a Tier-4 feature, not a class.
- **`pitch_type` codes**: Statcast occasionally emits `''` for un-classified pitches. Allow `''` in the assertion list; rare.
- **`base_state` encoding**: bitmask makes it trivial to query "runners on third" with `bitAnd(base_state, 4) > 0`.
- **`park_id == home_team`**: holds for v1. Park-relocation history (Athletics, Rays, etc.) is a v1.5 concern.
- **`FINAL` performance**: don't sprinkle `FINAL` everywhere — it forces merge during query. Use `argMax`/`OPTIMIZE TABLE` for hot paths in Phase 3.
- **No ON CLUSTER**: single-node ClickHouse, no cluster macros.
