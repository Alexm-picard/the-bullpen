## Triage rationale

This sweep coincided with the pre-season engineering pass that pulled the repo
back to a green, defensible baseline after a two-machine merge (desktop ML work +
laptop frontend redesign). Triage prioritised **correctness + green-CI floor**
over feature polish, because everything downstream (registration, the season
operate, the hiring story) builds on a trustworthy CI signal. The biggest finding
— both CI pipelines had been red and unwatched for ~6 days — was the forcing
function: closing it surfaced the rest (SpotBugs debt, type-safety gaps, a raw-SQL
leak, an un-validated detection chain). Each selected item carries a real
before/after on the same instrument, not "feels better." Pure feature/UX
observations (ops pagination, e2e harness, the un-wired live-poller) are deferred
to the in-season sweep — they're not on the correctness-floor critical path.

## Per-item notes

### 1–2. CI red-and-unwatched → green

- **Observation source**: `observations.md` 2026-05-30 (ci).
- **What broke**: backend `test` had been red since 2026-05-24 and training
  `lint-and-type` since the 2026-05-30 desktop merge — neither watched. The
  dev/prod split (ADR-0006) means desktop commits never ran the full gate locally.
- **Approach**: calibrated the gates to the codebase's conventions and FIXED the
  genuine findings (a dead store, 3 null-guards, an import-sort), excluding only
  documented framework/record/research noise.
- **Measurement**: `./gradlew build` SpotBugs count; `uv run ruff check` +
  `pyright` error counts; the GitHub Actions conclusion.
- **Result**: SpotBugs 120→0; ruff 56→0; pyright 98→0; `main` red→green across
  backend, training, contract.
- **Lessons**: a red gate nobody watches is worse than no gate. **Mitigation
  candidate (next sweep): a pre-push hook running the fast gates, or a
  branch-protection required-status check so red `main` is impossible.**

### 3. Schemathesis caught two real `500`s

- **Observation source**: new (S1f, added springdoc + a contract CI job).
- **What broke**: a non-numeric path variable and a firewall-rejected header each
  returned `500` instead of `400` on malformed input.
- **Measurement**: `schemathesis run /v3/api-docs --checks not_a_server_error`,
  30 examples/op.
- **Result**: 2 server errors → 0; 1283/1283 generated cases pass. Fixes:
  `MethodArgumentTypeMismatchException` handler + `HttpStatusRequestRejectedHandler`.
- **Lessons**: property-based API fuzzing earns its keep immediately — these were
  real defects no example-based test had.

### 4. Full TS strict

- **Observation source**: `observations.md` 2026-05-25 (frontend type-safety).
- **What broke**: the frontend compiled with `strict` entirely off — `tsc -b`
  showed 67 latent `noUncheckedIndexedAccess` issues once enabled.
- **Result**: 67→0, fixed with real guards (fail-loud invariants for dynamic
  lookups, bounded-loop locals), not blanket `!`. 399 tests stayed green.

### 6. Leakage checks proven to have teeth

- **What wasn't good enough**: the 4 leakage tests (rule 10) could have been
  vacuous — passing regardless of input is "theatre."
- **Result**: 3 Hypothesis properties over random folds prove the CLEAN pipeline
  is invariant to future labels AND a deliberately-LEAKY one is caught — the
  check distinguishes leak from no-leak.

### 10. Drift chain validated end-to-end

- See the full drill postmortem
  ([`drill-2026-05-30-induced-battedball-drift.md`](../postmortems/drill-2026-05-30-induced-battedball-drift.md)).
  Real PSI/ECE math + production alert + retrain trigger; the chain stopped
  exactly at the human-gated promotion (rule 6).
