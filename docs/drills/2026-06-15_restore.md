## Summary

The disaster-recovery path works. A single self-contained `--from-r2` run fetched the latest offsite
set from Cloudflare R2, restored both ClickHouse (15 tables, 7.67M-row `pitches`) and the SQLite
registry into a throwaway scratch instance, and booted both app profiles (`api` + `worker`) to
actuator `UP` against the restored data. It used **only** the three connection-config env vars the
systemd timer already provides (`BULLPEN_OFFSITE_REMOTE`, `RCLONE_CONFIG`, the scratch CH password)
and **no** toy/contract/admin overrides and **no** `CH_DB`/`FETCH_DIR` tricks - so the PASS is the
real, reproducible drill, not a hand-tuned one.

## The road here (why this PASS is trustworthy)

Rule 8 stayed open for weeks not because the backup was bad - it was restorable the whole time - but
because running the real thing kept surfacing the next harness or storage defect that code review did
not. Every one of the seven fixes was a harness, offsite-storage, or boot-config defect; **not once
was the backup itself bad.**

- **#75** - store the offsite ClickHouse backup as one tar object, not ~64k tiny objects (the
  2026-06-13 FAIL: a 64k-object `rclone copy` did not round-trip).
- **#79** - offsite push via `rclone copyto`, not `rcat` (R2 returned `NotImplemented` on the
  streaming upload).
- **#92 / #94** - drill harness: disk-backed `mktemp` staging (not tmpfs `/tmp`), `CH_DB=default`
  (the app's real DB), `1 <= scratch <= live` registry check, on-entry `mkdir -p` after cleanup.
- **#96** - `rclone copy --multi-thread-streams 1` + a content gate (`tar -tf` + `*.bin` count); the
  default multi-threaded download produced a right-size but corrupt 2.6 GB object.
- **#97** - the recurring "hollow restore": `find ... | grep -q .` under `pipefail` was a SIGPIPE
  false-fail on large result sets (grep closes the pipe on the first match, `find` dies 141,
  `pipefail` propagates). Replaced with `find -print -quit` + command substitution. The backup was
  always restorable; the check was lying.
- **#109** - boot step (gap 1): `boot_profile` launched the scratch JVM without the toy-ONNX artifact
  paths, which `ToyBattedBallInference` (`@Profile("api")`, `@PostConstruct`) resolves relative to the
  JVM CWD. Now passes absolute, script-location-derived `--bullpen.inference.toy.artifacts-dir` +
  `--bullpen.inference.contract-path` (the way the systemd unit does), plus an api-scoped preflight.
- **#110** - boot step (gap 2): the `#109`-harness run (`bcdz31bc2`) got past the toy ONNX and hit the
  next fail-closed bean - `SecurityConfig.adminUser` throws on a blank `bullpen.admin.basicauth`. Now
  passes a throwaway `--bullpen.admin.basicauth=drill:drill`. An audit of every api `@Value`-without-
  default + `IllegalStateException`-on-blank confirmed this was the last gap (see Findings).

- `warm-up failed; readiness stays DOWN` in `boot-api.log` - `WarmupReadiness` tries to warm the

## Findings

1. **The backup was never the problem - the harness was.** All seven fixes were harness,
   offsite-storage, or boot-config defects; the backup round-tripped from the start. This is the
   entire value of the drill: an untested backup is not a backup, and the untested-ness was hiding
   bugs that would have masqueraded as data loss in a real incident.
2. **A scratch-boot drill must supply the same env the systemd unit does.** The last two gaps (#109
   toy paths, #110 admin basic-auth) were both api beans that fail closed on env config production
   provides via `/etc/default/bullpen`. The canonical list to supply is exactly the fail-closed
   beans: every api `@Value`-without-default + `IllegalStateException`-on-blank was audited -
   `clickhouse.password` + `admin.basicauth` are the only two that fail closed; `R2ArchiveClient` is
   `@ConditionalOnExpression`-gated off when `S3_ENDPOINT_URL` is unset; `WarmupReadiness`'s
   absent-champion failure only downs the readiness probe, not the health gate.
3. **`find | grep -q` / `find | head` under `set -o pipefail` is a SIGPIPE false-fail trap** on large
   result sets (#97). Use `find -print -quit` + command substitution, or `tail` (reads to EOF). The
   same class was fixed in `usb-backup.sh` (`sort -r | head` -> `sort | tail`).
