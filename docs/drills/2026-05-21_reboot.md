## Method

Operator chose to power down the Windows desktop overnight and use the morning boot-up as the drill's recovery leg. Better Stack monitor will be muted before shutdown to avoid Discord alert spam through the night.

The boot-up sequence has manual steps inherent to WSL2:

1. Windows desktop powers on
2. Operator logs in
3. Operator opens a WSL2 terminal (this triggers the WSL2 distro to start — WSL2 does NOT auto-start without a Windows-side scheduled task; opening a terminal is the manual trigger)
4. systemd boots inside WSL2 → all units that were enabled fire
5. Docker daemon starts → containers with restart policies come back
6. cloudflared registers connectors with Cloudflare edge
7. bullpen-api + bullpen-worker start, listen on 8080/8081
8. Health probes go green
9. Better Stack monitor recovers

## Pass/fail criteria

- **Pass**: items 1–11 all green within 5 min of opening the WSL2 terminal post-boot
- **Partial pass**: 1–10 green within 5 min, item 11 deferred (acceptable — that's just Better Stack UI work)
- **Fail**: any of items 5, 6, or 9 don't recover, or item 10 takes >5 min
- On fail: capture the failing service's journal, write up the gap, add a remediation task; do NOT call Phase 0 done
